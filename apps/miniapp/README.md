# Coze AI 微信小程序助手

## 项目简介

这是一个基于微信小程序云开发和Coze AI的聊天助手，使用了以下技术栈：

- 微信云开发（云函数、数据库、存储）
- COZE API（AI对话引擎）
- 数据库实时推送（实时响应）

## 最新架构更新

### 2025-03-08更新：新架构采用云开发数据库实时推送

我们对架构进行了重大优化，将原来的"云函数 -> 云托管 -> Coze API"模式更改为"云函数 -> Coze API + 数据库实时推送"模式，主要好处：

1. **解决超时问题**：云函数超时设置为900秒（15分钟），足够处理大多数AI请求
2. **实时响应体验**：通过数据库实时推送功能，用户不需要等待整个请求完成
3. **更简单的架构**：无需维护云托管服务，降低了部署和维护复杂度
4. **更好的错误处理**：所有状态变化都记录在数据库中，便于跟踪和恢复

## 功能特点

- **长时间AI对话**：支持最长900秒的AI对话生成时间
- **实时响应**：通过数据库实时推送，实时显示AI回复
- **对话历史保存**：自动保存对话历史，支持会话上下文
- **错误处理**：完善的错误处理和重试机制
- **实时状态更新**：对话状态实时可见

## 快速开始

### 1. 部署云函数

1. 在微信开发者工具中，右键点击`cloudfunctions/cozeAPIv2`
2. 选择"上传并部署：云端安装依赖"

### 2. 初始化数据库

1. 在微信开发者工具中，右键点击`cloudfunctions/initDatabase`
2. 选择"上传并部署：云端安装依赖"
3. 在云开发控制台中，手动触发一次`initDatabase`云函数

### 3. 设置Coze API密钥

1. 在`cloudfunctions/cozeAPIv2/index.js`中设置您的Coze API密钥和Bot ID
2. 重新部署云函数

### 4. 运行小程序

直接在微信开发者工具中编译运行小程序即可开始使用。

## 实现原理

1. **发起请求**：
   - 小程序调用`cozeAPIv2`云函数发送查询
   - 云函数将初始状态写入`conversations`集合
   - 云函数开始调用Coze API

2. **实时监听**：
   - 小程序使用`db.watch()`功能监听数据库文档变化
   - 当云函数收到AI回复后，更新数据库记录
   - 小程序收到数据变化通知，实时更新UI

3. **状态管理**：
   - 请求过程中有多种状态：pending, processing, completed, error
   - 每个状态变化都记录在数据库中，便于追踪

## 开发注意事项

1. **超时设置**：在小程序云开发控制台中确保`cozeAPIv2`云函数超时设置至少为300秒
2. **WebSocket连接限制**：单个小程序最多可以同时保持5个WebSocket连接
3. **调试技巧**：使用小程序的云开发调试功能可以实时查看云函数日志

## 常见问题

1. **云函数超时**：检查云开发控制台中的超时设置
2. **无法接收实时更新**：确认WebSocket连接状态，检查网络连接
3. **API调用失败**：检查API密钥和Bot ID是否正确设置
