# Task Card: scraper 采集数据关联 work_id

## 需求（What & Why）
**功能描述**: 修改快手 scraper 脚本，在采集到平台数据后，通过 platform_post_id 查询 zenithjoy.publish_logs 表找到 work_id，将采集指标（metrics）回填到 publish_logs。

**背景**: zenithjoy.publish_logs 表有 platform_post_id 字段和 metrics JSONB 字段。scraper 已能采集快手 workId（即 platform_post_id），两者需打通做跨平台作品绩效分析。

**不做什么**: 不修改 social_media_raw 的写入逻辑；不改其他平台 scraper。

## 成功标准

1. [ARTIFACT] scraper-kuaishou-v3.js 增加第二个 DB 连接（cecelia 数据库）
2. [BEHAVIOR] 采集到 workId 时，查询 zenithjoy.publish_logs WHERE platform_post_id = $1 AND platform = 'kuaishou'，若找到则更新 metrics JSONB
3. [BEHAVIOR] 未找到 publish_logs 记录时，不影响现有 social_media_raw 写入流程
4. [GATE] CI 全部通过
