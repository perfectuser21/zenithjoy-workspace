# Learning: CI Phase 2 — API ESLint + 垃圾清理

## 做了什么
给 API 加了 ESLint 检查，CI 加了 api-lint job，.gitignore 清理了遗留垃圾目录。

## 关键决策

1. **ESLint flat config 用 .mjs** — API 是 CommonJS（没有 `"type": "module"`），ESLint flat config 用 ESM import，必须用 `.mjs` 扩展名。

2. **双模式 ESLint 配置** — API 中混有 `.js`（CommonJS）和 `.ts` 文件，需要分别配置 globals 和规则。

3. **Coverage gate 推迟** — `--coverage.enabled` 需要 `@vitest/coverage-v8` 依赖，不值得为了 CI 输出 coverage 而给每个 app 加一个 devDependency。留到 Phase 3 统一处理。

## 踩坑

### 根本原因
ESLint flat config 中 `js.configs.recommended` 对所有文件设了 `no-undef: error`。TypeScript 文件块可以覆盖掉，但混杂的 `.js` 文件如果没有单独配置 Node.js globals（`require`、`module`、`process`），会报大量 `no-undef` 错误。

### 下次预防
给混合 JS/TS 的 Node.js 项目写 ESLint flat config 时，一定要检查是否有 `.js` 文件，给它们单独加 CommonJS + Node globals。
