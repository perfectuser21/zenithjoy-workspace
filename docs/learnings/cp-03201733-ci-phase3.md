# Learning: CI Phase 3 — Coverage + Sensitive File Scan

## 做了什么
安装 @vitest/coverage-v8，CI 输出 coverage 报告。加 sensitive file scan。

## 关键决策

1. **@vitest/coverage-v8 版本匹配** — API 的 vitest 是 3.x，dashboard 是 4.x。coverage 包必须匹配 vitest 主版本号，不能用 latest。API 用 `@vitest/coverage-v8@3`。

2. **Coverage 不设阈值** — 当前覆盖率较低（dashboard ~50%，API ~10%），先输出报告建立 baseline，不设 fail 阈值。等覆盖率逐步提升后再加 gate。

## 踩坑

### 根本原因
`@vitest/coverage-v8` 的主版本号必须与 `vitest` 一致。monorepo 中不同 workspace 可能有不同 vitest 版本，导致 latest coverage 包与某些 workspace 不兼容。

### 下次预防
安装 vitest 生态包时，先 `npm ls vitest` 确认版本，再用 `@vitest/xxx@主版本号` 安装。
