# Task Card: scraper 指标同步到 zenithjoy.works（通过 work_id 关联）

## 需求（What & Why）

**功能描述**：创建 `scripts/sync-scraper-to-works.sh` 同步脚本，将 `social_media_raw` 中 scraper 采集的平台指标（views、likes 等）同步到 `zenithjoy.publish_logs`，通过 `title+platform` 匹配建立关联，从而让 `zenithjoy.works` 可查到发布后的实际数据。

**背景**：
- Scraper 采集数据写入 `social_media_raw.content_master` + `content_snapshots`（独立 DB）
- `zenithjoy.publish_logs` 存发布记录（work_id、platform），但没有指标数据
- 两个 DB 之间没有自动同步机制
- 通过 `platform + title` 匹配内容，将 scraper 指标写入 `publish_logs.response` JSONB

**不做什么**：不修改 scraper 脚本内部逻辑（保持单一职责），不做实时同步（批量脚本按需运行）。

## 成功标准

1. [ARTIFACT] `scripts/sync-scraper-to-works.sh` 存在

5. [BEHAVIOR] 脚本在找不到匹配时优雅跳过，不报错退出

## 实现方案

**文件**：`scripts/sync-scraper-to-works.sh`

**流程**：
```
1. 查 social_media_raw: SELECT cm.title, cm.platform, cs.views, cs.likes, ...
   FROM content_master cm JOIN content_snapshots cs ON ...
   WHERE cm.platform = $PLATFORM AND cs.snapshot_date = CURRENT_DATE

2. 对每条记录，查 cecelia:
   SELECT pl.id FROM zenithjoy.publish_logs pl
   JOIN zenithjoy.works w ON pl.work_id = w.id
   WHERE pl.platform = $PLATFORM AND w.title ILIKE $TITLE
   LIMIT 1

3. 如找到匹配：
   UPDATE zenithjoy.publish_logs
   SET response = jsonb_set(COALESCE(response, '{}'), '{metrics}',
     '{"views": X, "likes": Y, "comments": Z, "synced_at": "..."}')
   WHERE id = $LOG_ID
```
