# Task Card: ZenithJoy Monorepo CI 统一化 Phase 1

## 需求（What & Why）
**功能描述**: 将 zenithjoy monorepo 的 CI 从"只检查 dashboard"升级为全模块覆盖，统一 TypeScript 配置，清理配置矛盾。
**背景**: zenithjoy 从多 repo 合并为 monorepo，但 CI 仍停留在单 repo 水平。API 完全不检查，测试不在 CI 中跑，TypeScript 配置不统一（API strict vs dashboard non-strict），API 同时有 Jest 和 Vitest 两套测试框架。需要对齐到 cecelia 的正规水平。
**不做什么**: 不做 cecelia 的 L1-L4 四层分层（那是为 Brain/Engine 定制的）；不加 PostgreSQL 集成测试；不加 contract drift 检查。

## 成功标准
> [ARTIFACT] 产出物 / [BEHAVIOR] 运行时行为 / [GATE] 门禁

1. [ARTIFACT] 存在 tsconfig.base.json，apps/dashboard 和 apps/api 的 tsconfig.json 继承它
2. [ARTIFACT] apps/api 只有一个测试框架（Vitest），Jest 配置已删除
3. [ARTIFACT] apps/api/Dockerfile 使用 node:20-alpine
4. [BEHAVIOR] CI 有 change detection — dashboard/api/geoai 变更分别触发对应检查
5. [BEHAVIOR] CI 跑测试 — dashboard 跑 vitest run，api 跑 vitest run
6. [GATE] CI gate job 使用 if: always() + 逐项检查模式
7. [GATE] 所有现有测试通过

- [ ] [ARTIFACT] apps/api 无 Jest 配置文件（jest.config.js 已删除）
  Test: manual:bash -c "test ! -f apps/api/jest.config.js && echo PASS || echo FAIL"

- [ ] [ARTIFACT] apps/api/Dockerfile 使用 node:20
  Test: manual:bash -c "grep -q 'node:20' apps/api/Dockerfile && echo PASS || echo FAIL"

- [ ] [BEHAVIOR] CI workflow 包含 change detection job
  Test: manual:bash -c "grep -q 'Detect Changes' .github/workflows/ci.yml && echo PASS || echo FAIL"

- [ ] [BEHAVIOR] CI workflow 包含 api lint+typecheck+test jobs
  Test: manual:bash -c "grep -q 'api-lint\|api-typecheck\|api-test' .github/workflows/ci.yml && echo PASS || echo FAIL"

- [ ] [BEHAVIOR] CI workflow 包含 dashboard test job
  Test: manual:bash -c "grep -q 'dashboard-test\|test:' .github/workflows/ci.yml && echo PASS || echo FAIL"

- [ ] [GATE] 所有现有测试通过

## 实现方案
**要改的文件**:
- `tsconfig.base.json` (新建) — 共享 TS 配置
- `apps/api/tsconfig.json` — 继承 base，移除 jest types
- `apps/dashboard/tsconfig.json` — 继承 base，开启 strict
- `apps/api/package.json` — Jest→Vitest，添加 typecheck/lint 脚本
- `apps/api/jest.config.js` (删除) — 去重
- `apps/api/Dockerfile` — node:18→20
- `.github/workflows/ci.yml` — 全面重写
- `apps/dashboard/src/components/video-generation/VideoPreview.tsx` — strict 类型修复
- `apps/dashboard/src/pages/ContentPublish.tsx` — strict 类型修复
- `apps/dashboard/src/pages/RawDataTable.tsx` — strict 类型修复

**Scope 锚定**: 配置层 + CI 层，不改业务逻辑
