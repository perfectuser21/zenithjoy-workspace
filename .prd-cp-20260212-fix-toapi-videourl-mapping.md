# PRD: 修复 ToAPI 视频 URL 映射问题

## 问题描述

用户反馈：视频生成任务完成后，右侧内容消失，看不到视频在哪。

## 根本原因

通过浏览器 console 日志分析发现：

```javascript
{
  task_id: "video_c913b29f-0991-480b-ad72-93af1be47beb",
  status: "SUCCESS",
  fail_reason: "https://files.toapis.com/flow/713b2892-f218-45a9-9046-53c3394e52f2.mp4"
}
```

**问题**：
1. ToAPI API 把视频 URL 放在 `fail_reason` 字段（不是标准的 `video_url` 字段）
2. 当前代码只映射 `video_url` 字段：`videoUrl: task.video_url`
3. 导致 `videoUrl` 为 undefined
4. VideoPreview 组件检查 `if (!task.videoUrl) return null;`
5. 结果：任务完成后不显示视频预览

## 解决方案

在 `apps/dashboard/src/api/platforms/toapi.ts` 中：

1. **添加 fail_reason 字段到接口**：
```typescript
interface ToAPITask {
  // ... existing fields
  fail_reason?: string;  // ToAPI 的 bug：视频 URL 在这里
}
```

2. **修改 mapToUnifiedTask() 逻辑**：
```typescript
// 视频 URL 映射：优先使用 video_url，如果不存在且 fail_reason 是 URL，使用 fail_reason
let videoUrl = task.video_url;
if (!videoUrl && task.fail_reason?.startsWith('http')) {
  videoUrl = task.fail_reason;
}

return {
  // ...
  videoUrl,
  // ...
};
```

这是 ToAPI 的 API bug，但我们需要在客户端做兼容处理。

## 影响范围

- 文件：`apps/dashboard/src/api/platforms/toapi.ts`
- 影响：所有 ToAPI 视频生成任务完成后的视频预览
- 严重程度：**Critical** - 用户看不到生成的视频

## 验收标准

1. 修改 ToAPITask 接口，添加 `fail_reason` 字段
2. 修改 mapToUnifiedTask() 方法，支持从 `fail_reason` 读取视频 URL
3. 更新测试，覆盖 fail_reason 场景
4. 手动测试：提交视频生成任务，等待完成，验证显示视频预览和下载按钮

## 测试场景

1. 提交视频生成任务
2. 等待任务完成（不超时）
3. 验证右侧显示视频预览界面
4. 验证视频可以播放
5. 验证"下载视频"和"生成新视频"按钮可用
