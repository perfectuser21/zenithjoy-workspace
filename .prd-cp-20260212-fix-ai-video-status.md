## PRD - 修复 AI 视频生成任务状态显示问题

**需求来源**: 用户反馈 - 点击"开始生成视频"后页面空白，没有进度显示

**问题分析**:
1. 用户点击"开始生成视频"按钮后，页面右侧完全空白
2. Console 显示错误：`Unknown task status: undefined`（重复 6 次）
3. Network 请求显示正在轮询 ToAPI: `GET https://toapis.com/v1/videos/generations/task_01KH8AZA64XK3CQX1ZFNTK2SFX`
4. TaskMonitor 组件检测到 task.status 为 undefined，返回 null 导致页面空白

**根本原因**:
ToAPI 返回的任务对象数据格式与前端类型定义不匹配：
- 前端期望：`status: 'queued' | 'in_progress' | 'completed' | 'failed'`
- ToAPI 实际返回：`status` 字段可能不存在或使用不同的字段名/值

**功能描述**:
1. 添加调试日志，打印 ToAPI 实际返回的数据结构
2. 根据 ToAPI 真实格式，添加数据映射层：
   - 如果字段名不同（如 `state` vs `status`），映射字段名
   - 如果状态值不同（如 `processing` vs `in_progress`），映射状态值
3. 添加容错处理：如果 status 仍然是 undefined，显示默认加载状态
4. 改进 TaskMonitor UI：显示有意义的加载提示而不是空白

**涉及文件**:
- `apps/dashboard/src/api/video-generation.api.ts` - 添加数据映射和调试日志
- `apps/dashboard/src/types/video-generation.types.ts` - 可能需要添加 ToAPI 原始格式类型
- `apps/dashboard/src/components/video-generation/TaskMonitor.tsx` - 改进错误状态显示

**成功标准**:
1. ✅ 点击"开始生成视频"后，页面显示任务监控组件
2. ✅ 显示任务状态（排队中/生成中/已完成/失败）
3. ✅ 如果有进度信息，显示进度条
4. ✅ Console 无 "Unknown task status" 错误
5. ✅ 用户能实时看到视频生成进度

**非目标**:
- ❌ 不修改 ToAPI 后端 API（只适配前端）
- ❌ 不改变现有的视频生成功能逻辑
- ❌ 不添加新的视频生成参数
