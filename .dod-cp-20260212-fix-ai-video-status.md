## 实施细节

### 数据映射

ToAPI 实际响应格式（待调试确认）：
```typescript
// 可能的格式 1: state 而非 status
{ id: "task_xxx", state: "processing", ... }

// 可能的格式 2: 不同的状态值
{ id: "task_xxx", status: "pending", ... } // 而非 "queued"
```

映射层需处理：
1. 字段名差异（state → status）
2. 状态值差异（processing → in_progress）
3. undefined 兜底（显示 "loading" 状态）
