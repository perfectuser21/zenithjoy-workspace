# PRD: ZenithJoy Monorepo 整合

## 背景

zenithjoy 仓库下有多个独立子项目（各自有独立 .git 或游离在 workspaces 外），每次开发需要跨多个目录，维护成本高。

## 目标

将以下子项目全部纳入 zenithjoy 主 monorepo 统一管理：
- `workspace/` — Node.js 前端
- `geoai/` — Astro + Cloudflare Workers
- `creator/` — Python 脚本 + n8n workflow（独立 git repo）
- `workflows/` — Node.js 子包集合
- `JNSY-Label/` — Python + Docker

## 方案

### 目录结构调整
```
zenithjoy/
├── package.json          ← workspaces 扩展覆盖所有 Node.js 项目
├── apps/
│   ├── api/              ← 原有，保留
│   ├── dashboard/        ← 原有，保留
│   ├── workspace/        ← 从 workspace/ 移入
│   └── geoai/            ← 从 geoai/ 移入
├── packages/
│   └── workflows/        ← 从 workflows/ 移入（含子包）
└── services/
    ├── creator/          ← 从 creator/ 移入（Python，删除子 .git）
    └── jnsy-label/       ← 从 JNSY-Label/ 移入（Python + Docker）
```

### 操作清单
1. 删除 `creator/.git`（子 git repo → 纳入主 repo）
2. 移动目录：workspace → apps/workspace，geoai → apps/geoai
3. 移动目录：creator → services/creator，JNSY-Label → services/jnsy-label
4. 移动目录：workflows → packages/workflows
5. 更新根 `package.json` workspaces 覆盖所有 Node.js 项目
6. 更新 `.gitignore` 排除 Python 缓存等
7. 更新根 CI（.github/workflows/）适配新结构

## 成功标准

- 所有子项目代码在主 repo 可见（git status 显示正确）
- `npm install`（根目录）正常安装所有 Node.js 依赖
- Python 项目结构完整，requirements.txt 存在
- CI 配置正确，能识别各子项目
- 旧的游离目录不再存在（已移动或重命名）
