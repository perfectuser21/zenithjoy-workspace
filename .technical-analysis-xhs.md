# 小红书自动发布技术可行性分析

## 分析时间

2026-02-12（Exploratory 模式）

## 核心结论

✅ **技术可行性：高** - 基于今日头条和抖音的成功经验，小红书发布器完全可行

## 架构方案（复用成功模式）

```
NAS 存储 (xhs-post-*)
    ↓
Mac mini 调度器 (scheduler-xhs.sh)
    ↓ Base64 + scp
Windows PC Playwright
    ↓ CDP 连接
小红书创作平台 ✅
```

### 为什么选择这个架构？

| 组件 | 理由 |
|------|------|
| NAS 存储 | 统一的内容源，多平台共享 |
| Mac mini 调度器 | 访问 NAS，协调发布流程 |
| Windows PC | 浏览器已登录，CDP 连接避免重复登录 |
| Playwright | 自动化发布，监听 API 判断成功 |

## 内容类型（2 种）

### 1. 图文笔记

**必需字段**：
- `title` - 标题（必需）
- `content` - 正文内容（必需）
- `images[]` - 图片数组（1-9张）

**可选字段**：
- `tags[]` - 话题标签（如 #美食#）
- `location` - 地理位置

**NAS 目录结构**：
```
creator/output/YYYY-MM-DD/xhs-post-1/
├── type.txt          # "image"
├── title.txt         # 标题
├── content.txt       # 正文
├── image-1.jpg       # 图片1
├── image-2.jpg       # 图片2（可选）
└── ...               # 最多9张
```

### 2. 视频笔记

**必需字段**：
- `title` - 标题（必需）
- `content` - 描述文字（必需）
- `video` - 视频文件（必需）
- `cover` - 视频封面（必需 ⚠️ 基于抖音教训）

**NAS 目录结构**：
```
creator/output/YYYY-MM-DD/xhs-post-2/
├── type.txt          # "video"
├── title.txt         # 标题
├── content.txt       # 描述
├── video.mp4         # 视频
└── cover.jpg         # 封面
```

### 2. API 监听

**需要识别的 API 端点**：

基于其他平台的经验，小红书可能的 API 端点：

| API 端点（推测） | 用途 | 判断成功的字段 |
|----------------|------|--------------|
| `/api/sns/web/v1/note` | 发布笔记 | `success: true` 或 `code: 0` |
| `/api/sns/web/v1/upload` | 上传图片/视频 | `url` 字段存在 |
| `/web/api/note/create` | 创建笔记 | `note_id` 字段存在 |

**验证方法**：
1. 打开浏览器开发者工具 - Network 标签
2. 手动执行一次发布流程
3. 筛选 XHR/Fetch 请求
4. 找到发布成功的 API 请求
5. 记录 URL、请求体、响应体

### 3. 成功标志

**多重验证机制**：

| 验证方式 | 可靠性 | 说明 |
|---------|--------|------|
| API 响应 | ⭐⭐⭐⭐⭐ | 最可靠，有明确的 success/code 字段 |
| URL 跳转 | ⭐⭐⭐⭐ | 发布后跳转到 `/content` 或 `/notes` |
| 页面元素 | ⭐⭐⭐ | 出现"发布成功"提示 |

**推荐策略**：
```javascript
// 优先级1: API 响应
if (apiResponse.success || apiResponse.code === 0) {
  return { success: true, noteId: apiResponse.data.note_id };
}

// 优先级2: URL 跳转
if (currentUrl.includes('/content') || currentUrl.includes('/notes')) {
  return { success: true };
}

// 优先级3: 超时失败
return { success: false, error: 'Timeout' };
```

### 4. 等待时间

**基于其他平台的经验**：

| 操作 | 建议等待时间 | 原因 |
|------|------------|------|
| 页面加载 | 2-3秒 | 等待 JS 加载完成 |
| 图片上传 | 3-5秒/张 | 取决于图片大小 |
| 视频上传 | 10-30秒 | 取决于视频大小 |
| 点击按钮 | 1-2秒 | 等待动画和 JS 响应 |
| 发布完成 | 5-10秒 | 等待 API 响应和页面跳转 |

### 5. 文件上传方式

**两种可能的方式**：

**方式1: Input File（推荐）**
```javascript
const fileInput = await page.locator('input[type="file"]');
await fileInput.setInputFiles(['/path/to/image.jpg']);
```

**方式2: File Chooser（备选）**
```javascript
const [fileChooser] = await Promise.all([
  page.waitForEvent('filechooser'),
  page.click('button.upload')
]);
await fileChooser.setFiles(['/path/to/image.jpg']);
```

**验证方法**：
1. 检查上传按钮的 HTML 结构
2. 确认是否有隐藏的 `<input type="file">`
3. 如果有，用方式1；如果没有，用方式2

## 技术风险评估

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|-------|------|---------|
| 页面结构变化 | 中 | 高 | 使用多个备选选择器 |
| API 端点变化 | 低 | 高 | 监听多个可能的端点 |
| 登录态失效 | 低 | 中 | CDP 连接已登录的浏览器 |
| 上传失败 | 中 | 中 | 检查文件大小和格式，增加重试 |
| 话题标签格式 | 低 | 低 | 测试 #话题# 格式 |
