# Task Card: System Registry v2.0 统一架构

## 需求（What & Why）
**功能描述**: 将现有单体 feature-registry.yml 重构为两层结构：system-registry.yml（总表）+ features/*.yml（每系统详情），每个 feature 融合 code/docs/skills/apis/tests/ci/runtime 全部信息。
**背景**: 现有 feature-registry.yml 是单文件扁平结构，缺少 system 层级分组，且缺少 skills/apis/runtime 等维度。v2.0 与 Cecelia 格式对齐（schema_version: "2.0.0"）。
**不做什么**: 不修改 registry.json（保留作为脚本运行时读取用）；不修改 api-registry.yml；不新增自动化脚本。

## 成功标准
> [ARTIFACT] docs/registry/system-registry.yml 总表文件
> [ARTIFACT] docs/registry/features/{auth,media,publisher,crm,ai-employee,ai-video}.yml 6 个详情文件
> [BEHAVIOR] 旧 feature-registry.yml 被删除

## 验收条件（DoD）

- [ ] [ARTIFACT] docs/registry/system-registry.yml 存在且 schema_version 为 "2.0.0"，包含 6 个 system 定义
  Test: manual:node -e "const y=require('js-yaml').load(require('fs').readFileSync('docs/registry/system-registry.yml','utf8')); console.assert(y.schema_version==='2.0.0'); console.assert(y.systems.length===6); console.log('PASS')"

- [ ] [ARTIFACT] docs/registry/features/ 目录下有 6 个 .yml 文件
  Test: manual:bash -c "test $(ls docs/registry/features/*.yml | wc -l) -eq 6 && echo PASS || echo FAIL"

- [ ] [BEHAVIOR] publisher.yml 包含 12 个 feature（8 平台 + probe/alert/keepalive/registry），每个含 code/docs/skills/apis/tests/ci/runtime 段
  Test: manual:node -e "const y=require('js-yaml').load(require('fs').readFileSync('docs/registry/features/publisher.yml','utf8')); console.assert(y.features.length===12); y.features.forEach(f=>{['code','docs','tests','ci','runtime'].forEach(k=>console.assert(k in f,k+' missing in '+f.feature_id))}); console.log('PASS')"

- [ ] [BEHAVIOR] feature-registry.yml 已被删除
  Test: manual:bash -c "test ! -f docs/registry/feature-registry.yml && echo PASS || echo FAIL"

- [ ] [ARTIFACT] system-registry.yml 底部 summary 包含 total_systems=6, total_features=27
  Test: manual:node -e "const y=require('js-yaml').load(require('fs').readFileSync('docs/registry/system-registry.yml','utf8')); console.assert(y.summary.total_systems===6); console.assert(y.summary.total_features===27); console.log('PASS')"
