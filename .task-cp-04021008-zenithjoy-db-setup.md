# Task Card: 建 zenithjoy DB + 3张核心表 + 爬虫改写直传美国

## 需求（What & Why）

**功能描述**:
1. 在 cecelia DB 的 zenithjoy schema 建 daily_snapshots 表（works/publish_logs 已存在）
2. 新增 snapshots API 路由（POST ingest + GET 查询）
3. 改8个平台爬虫脚本，采完直接 POST 到美国 API，不写西安本地
4. 删西安 social_media_raw 的8张 daily_snapshots 空表
5. 前端 platform-data.api.ts 指向新 /api/snapshots 接口

**背景**: ZenithJoy 内容数据架构重构。原来数据分散两台机器，结构不统一。
新架构：所有数据统一在美国 zenithjoy schema，西安只是 worker，采完直传美国。

**不做什么**:
- 不改 works / publish_logs 现有结构
- 不做 Day1/Day3/Day7 对齐计算（数据层先做）
- 不改 CI 配置

## 成功标准

1. zenithjoy.daily_snapshots 表存在，有正确列和唯一索引
2. POST /api/snapshots/ingest 能写入数据返回 200
3. GET /api/snapshots/:platform 能返回数据列表
4. GET /api/snapshots/work/:workId 关联 publish_logs 返回跨平台数据
5. 8个平台爬虫脚本末尾有 POST ingest 逻辑
6. 西安 social_media_raw 的8张 daily_snapshots 空表已删除
7. 前端 platform-data.api.ts 指向 /api/snapshots 接口
