# PRD - AI 视频生成历史记录功能

**需求来源**:
> "你应该给我一个单独的页面。就是你以后刷新了也不会变的话的东西啊。我你这现在是页面一刷新，之前的历史人物都都都没了，或者是比如说我正在创建，然后页面一刷新没了找不到了，你这不是搞笑吗"

**核心问题**:
1. ❌ 页面刷新后，所有历史生成记录消失
2. ❌ 正在生成的任务刷新后找不到，无法恢复
3. ❌ 没有独立的历史页面查看所有生成记录

**功能描述**:

实现完整的 AI 视频生成历史记录系统，包括：

### 1. 数据持久化
- 所有视频生成任务自动保存到数据库（PostgreSQL）
- 记录完整生成参数和状态
- 支持状态实时更新

### 2. 历史记录页面
- 独立路由：`/ai-video/history`
- 表格展示所有历史记录（分页）
- 筛选器：全部 | 成功 | 失败 | 进行中
- 每条记录显示：
  - 提示词（截断显示，hover 显示完整）
  - 状态标签（颜色区分）
  - 生成时间
  - 视频预览（completed 状态）
  - 操作按钮：下载、删除
- 点击记录弹窗显示详情

### 3. 任务恢复机制
- 页面加载时自动检测进行中的任务
- 从数据库恢复任务状态
- 继续轮询直到完成

### 4. 当前页面改造
- 提交任务时保存到数据库
- 右上角添加"查看历史"按钮
- 完成后自动跳转到历史页面（可选）

### 数据库 Schema
```sql
CREATE TABLE ai_video_generations (
  id VARCHAR(255) PRIMARY KEY,
  platform VARCHAR(50) NOT NULL,
  model VARCHAR(100) NOT NULL,
  prompt TEXT NOT NULL,
  status VARCHAR(50) NOT NULL,
  progress INT DEFAULT 0,
  video_url TEXT,
  error_message TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  completed_at TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

  -- 生成参数
  duration INT,
  aspect_ratio VARCHAR(20),
  resolution VARCHAR(20),

  INDEX idx_status (status),
  INDEX idx_created_at (created_at DESC)
);
```

**API 端点**:
- `POST /api/ai-video/generate` - 创建任务（同时保存到数据库）
- `GET /api/ai-video/history` - 查询历史记录（分页、筛选）
- `GET /api/ai-video/task/:id` - 查询单个任务状态
- `DELETE /api/ai-video/task/:id` - 删除历史记录
- `GET /api/ai-video/active` - 获取进行中的任务（用于恢复）

**成功标准**:

1. ✅ 提交新任务后立即保存到数据库
2. ✅ 刷新页面后历史记录仍然存在
3. ✅ 正在生成的任务刷新后自动恢复并继续轮询
4. ✅ 历史页面可以查看、筛选、下载、删除记录
5. ✅ 视频预览正常显示（已修复 fail_reason 映射问题）
6. ✅ 手动测试：
   - 提交任务 → 刷新页面 → 任务继续生成
   - 查看历史页面 → 所有记录正常显示
   - 筛选功能正常工作
   - 下载视频可用

**非目标**:

- ❌ 不实现多用户隔离（暂时单用户）
- ❌ 不实现视频本地存储（仅存储 ToAPI 返回的 URL）
- ❌ 不实现视频编辑功能
- ❌ 不实现批量操作
- ❌ 不实现导出功能

**技术栈**:

- 数据库：PostgreSQL
- 后端：Node.js + Express（现有架构）
- 前端：React + TypeScript + TanStack Query
- 状态管理：React Hooks + localStorage（兜底）

**优先级**:

- P0：数据库表 + 保存 API
- P0：刷新恢复机制
- P1：历史页面基础功能
- P2：删除、筛选等增强功能
