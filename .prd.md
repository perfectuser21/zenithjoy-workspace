## PRD - 修复 AI 视频生成页面空白问题

**需求来源**: 用户反馈 - 点击"开始生成视频"后页面空白，没有任何提示

**问题分析**:
1. 用户点击"开始生成视频"按钮后，页面右侧完全空白
2. 代码逻辑：
   - 点击按钮后，pageState 设置为 'generating'
   - 然后调用 createVideoGeneration API (异步)
   - API 返回后设置 taskId
   - 只有当 `pageState === 'generating' && taskId` 时才显示 TaskMonitor
3. **根本原因**：在 API 调用期间（约 1-3 秒），pageState 已经是 'generating' 但 taskId 还是 null，此时什么都不显示，导致页面空白

**功能描述**:
在 API 调用期间显示"正在提交任务..."的加载提示，避免页面空白

**技术方案**:
修改 `AiVideoGenerationPage.tsx` 中的渲染逻辑：
```tsx
{pageState === 'generating' && (
  <>
    {taskId ? (
      <TaskMonitor taskId={taskId} ... />
    ) : (
      <LoadingIndicator message="正在提交任务..." />
    )}
  </>
)}
```

**涉及文件**:
- `apps/dashboard/src/pages/AiVideoGenerationPage.tsx` - 添加 API 调用期间的加载提示

**成功标准**:
1. ✅ 点击"开始生成视频"后，立即显示"正在提交任务..."加载动画
2. ✅ API 返回后，切换到 TaskMonitor 显示任务状态
3. ✅ 页面不再出现空白
4. ✅ 用户体验流畅，无等待黑洞

**非目标**:
- ❌ 不修改 API 调用逻辑
- ❌ 不修改 TaskMonitor 组件
- ❌ 不改变现有的状态管理
