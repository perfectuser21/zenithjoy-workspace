# PRD: 作品管理系统 API (CRUD)

## 📋 任务概述

这是 **Phase 1: 基础架构** 的第二部分。数据库 Migration 已完成，现在需要实现基础 API。

**目标**：为作品管理系统创建 RESTful API，支持作品的增删改查操作。

## 🎯 核心功能

### 1. Works API (作品 CRUD)

5 个端点：
- GET /api/works - 获取作品列表（支持筛选、分页）
- GET /api/works/:id - 获取单个作品
- POST /api/works - 创建作品
- PUT /api/works/:id - 更新作品
- DELETE /api/works/:id - 删除作品

### 2. Field Definitions API (字段定义)

4 个端点：
- GET /api/fields - 获取所有字段定义
- POST /api/fields - 创建字段
- PUT /api/fields/:id - 更新字段
- DELETE /api/fields/:id - 删除字段

### 3. Publish Logs API (发布日志)

3 个端点：
- GET /api/works/:workId/publish-logs - 获取作品的发布日志
- POST /api/works/:workId/publish-logs - 创建发布日志
- PUT /api/publish-logs/:id - 更新发布状态

## 📐 技术架构

- **语言**: TypeScript
- **框架**: Express.js
- **数据库**: PostgreSQL (zenithjoy schema)
- **验证**: Zod (schema validation)
