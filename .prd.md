# PRD: 飞书登录改走 N8N Webhook

## 背景

当前飞书登录走单独的 feishu-auth-backend（US:3002），导致架构不一致：
- N8N 是唯一后端，但飞书认证却依赖一个额外的 Node.js 服务
- Dev 和 Prod 环境都需要额外配置代理到 US

## 目标

统一后端为 N8N：飞书登录也走 N8N webhook，去掉 feishu-auth-backend 依赖。

## 架构变更

**变更前：**
```
前端 → /api/feishu-login → US feishu-auth-backend (3002)
前端 → /api/n8n/         → N8N
前端 → /api/n8n-webhook/ → N8N webhook
```

**变更后：**
```
前端 → /api/n8n-webhook/webhook/feishu-login → N8N webhook
前端 → /api/n8n/                              → N8N REST API
前端 → /api/n8n-webhook/                      → N8N webhook
```

只有一个后端依赖：N8N。Dev 走 US N8N，Prod 走 HK N8N。

## 需求

### 前端变更
1. FeishuLogin.tsx：`/api/feishu-login` → `/api/n8n-webhook/webhook/feishu-login`
2. vite.config.ts：去掉 `/api/feishu-login` 单独代理
3. deploy/nginx.conf：去掉 `/api/feishu-login` 单独代理块

### 涉及文件
- `apps/dashboard/src/pages/FeishuLogin.tsx`
- `apps/dashboard/vite.config.ts`
- `deploy/nginx.conf`
