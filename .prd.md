# PRD: 数据库 Migration 脚本 - 作品管理系统

## 📋 任务概述

本次任务是 **ZenithJoy 作品管理系统**（参考 `.prd-works-management.md`）的 **Phase 1: 基础架构**。

创建数据库 migration 脚本，实现以下表结构：
1. `zenithjoy.works` - 作品表
2. `zenithjoy.publish_logs` - 发布日志表
3. `zenithjoy.field_definitions` - 字段定义表
4. 扩展 `platform_posts` 表，添加 `work_id` 字段

## 🎯 目标

为作品管理系统建立数据库基础，支持：
- 作品内容存储（文本/图文/视频/长文/音频）
- 多平台发布日志
- 自定义字段配置（类 Notion）
- 数据追踪关联（关联到 TimescaleDB）

## 📊 数据模型

### 1. zenithjoy.works（作品表）

**核心字段**：
- `id` (UUID) - 主键
- `title` (VARCHAR 500) - 标题
- `body` (TEXT) - 正文（Markdown）
- `content_type` (VARCHAR 20) - 类型（text/image/video/article/audio）
- `custom_fields` (JSONB) - 自定义字段

**时间字段**：
- `created_at`, `scheduled_at`, `first_published_at`, `updated_at`, `archived_at`

### 2. zenithjoy.publish_logs（发布日志表）

**核心字段**：
- `id`, `work_id` (外键), `platform`, `platform_post_id`, `status`, `published_at`

**约束**：
- 唯一约束：(work_id, platform)

### 3. zenithjoy.field_definitions（字段定义表）

**核心字段**：
- `id`, `field_name` (UNIQUE), `field_type`, `options` (JSONB)

### 4. platform_posts 扩展

**新增字段**：
- `work_id` (UUID) - 关联到 works 表
