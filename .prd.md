## PRD - 修复今日头条发布功能

**需求来源**: 用户反馈"今日头条发布一直发不出去，深度找一下问题解决这个事情"

**功能描述**:
通过大量 CDP 测试，发现今日头条发布的正确流程被误删除。原有的 `/home/xx/notion_toutiao_publisher.js` 脚本使用了错误的发布流程（直接点击"预览并发布"），导致无法成功发布。

**真实发布流程**（经测试验证）：
1. 填写标题和内容
2. 点击"预览并发布"进入设置页面
3. 选择"无封面"（避免图片上传要求）
4. 点击"预览"按钮（不是"预览并发布"）
5. 预览打开后，会出现新的"发布"按钮
6. 点击"发布"按钮完成发布

**涉及文件**:
- `/home/xx/notion_toutiao_publisher.js` - 主发布脚本，需要修复发布流程
- `/workflows/n8n/media/flow-notion到头条发布.json` - N8N workflow，通过 SSH 调用发布脚本

**成功标准**:
1. 脚本能够成功填写标题和内容（已验证可行）
2. 正确执行：进入设置 → 选择"无封面" → 点击"预览" → 点击"发布"
3. 页面跳转离开 `/publish` 路径，表示发布成功
4. N8N workflow 能够通过脚本成功发布到今日头条

**非目标**:
- 不处理图片上传功能（通过选择"无封面"绕过）
- 不处理定时发布功能
- 不处理多账号管理

**技术要点**:
- 使用 Chrome DevTools Protocol (CDP) 连接 Windows Chrome (100.97.242.124:19225)
- 使用 Object.getOwnPropertyDescriptor 绕过 React 受控输入
- 云端同步弹窗处理已在 commit 5064c1a 中修复，需保留
- 等待时间：进入设置 3s → 选择封面 1.5s → 点击预览 2.5s → 监控 30s
