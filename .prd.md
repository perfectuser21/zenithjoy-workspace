## PRD - Platform Scraper 凭据清理与路径修复

**需求来源**: workspace 整理，清理硬编码凭据，修复路径引用

**功能描述**:
1. vps_publisher.js 和 vps_scraper_dad.js 中硬编码的 IP/API Key 移到 .env
2. platform_scrape_wrapper.sh 路径引用修复（当前指向不存在的文件名）
3. scrape_and_sync.sh 路径引用修复（当前 hardcode /home/xx/）
4. 更新 .env.example 包含所有需要的环境变量

**涉及文件**:
- `workflows/platform-scraper/vps_publisher.js` - 移除 WINDOWS_IP, API_KEY, DASHBOARD_API 硬编码
- `workflows/platform-scraper/vps_scraper_dad.js` - 移除 WINDOWS_IP 硬编码
- `workflows/platform-scraper/platform_scrape_wrapper.sh` - 修复脚本查找路径
- `workflows/platform-scraper/scrape_and_sync.sh` - 修复 vps_scraper.js 路径引用
- `workflows/platform-scraper/.env.example` - 补充新增环境变量

**成功标准**:
- 所有脚本无硬编码 IP/Key/Password
- platform_scrape_wrapper.sh 能找到正确的 scraper 脚本
- .env.example 文档齐全
- CI 通过（lint + typecheck + build）

**非目标**:
- 不移动 /home/xx/ 下的 scraper-*-v3.js（N8N SSH 直接调用，移动需同步更新 N8N workflow）
- 不修改脚本的业务逻辑
- 不安装 platform-scraper 的 npm 依赖
