## PRD - Dashboard 架构独立化（去 Cecelia Core 依赖）

**需求来源**: ZenithJoy Dashboard 应独立运行，不依赖 Cecelia Core 后端

**功能描述**:

当前 Dashboard 所有 API 调用都指向 `core.zenjoymedia.media`（Cecelia Core），需要改为直连 N8N。

架构变更：
```
当前: Dashboard → core.zenjoymedia.media (Cecelia Core) → N8N
目标: Dashboard → Nginx /api → N8N REST API (localhost)
```

环境配置：
- 生产 (HK): Nginx → HK N8N (localhost:5679)
- 开发 (US): Vite proxy → US N8N (localhost:5679)

具体改动：

1. **InstanceContext** — 去掉 Core 实例检测，永远是 Autopilot 模式
2. **.env 文件** — VITE_API_URL 改为空（用 Nginx/Vite 代理），新增 VITE_N8N_API_URL
3. **API 文件** — `dashboard.api.ts` 和 `ai-employees.api.ts` 中 `VITE_API_URL + /api/v1/n8n-live-status/...` 改为 `/api/n8n/...`（走 Nginx 代理到 N8N REST API）
4. **Nginx 配置** — 添加 `/api/n8n/` → N8N REST API (5679) 的反代
5. **Vite 配置** — 开发环境 proxy 添加 `/api/n8n` → localhost:5679
6. **vite.config.ts** — 去掉 `@features/core` 对 cecelia 路径的引用
7. **scraping.api.ts** — N8N webhook URL 通过 Nginx 代理

**涉及文件**:
- `apps/dashboard/src/contexts/InstanceContext.tsx`
- `apps/dashboard/src/api/dashboard.api.ts`
- `apps/dashboard/src/api/ai-employees.api.ts`
- `apps/dashboard/src/api/scraping.api.ts`
- `apps/dashboard/.env.development`
- `apps/dashboard/.env.production`
- `apps/dashboard/vite.config.ts`
- `deploy/nginx.conf`
- `deploy/docker-compose.hk.yml`

**成功标准**:
- 代码中不再出现 `core.zenjoymedia.media` 或 `dev-core.zenjoymedia.media`
- 不再引用 `@features/core` 指向 cecelia 仓库路径
- isCoreInstance() 逻辑移除，永远是 Autopilot
- Nginx 配置有 /api/n8n/ 反代到 N8N
- CI 通过（lint + typecheck + build）

**非目标**:
- 不实现新的后端服务（accounts/publish/metrics 等 API 暂时保留，后续由 N8N workflow 提供）
- 不修改 N8N workflow 本身
- 不部署到 HK（只改代码和配置）
