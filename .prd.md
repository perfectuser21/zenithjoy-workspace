# PRD: 修复 API 测试失败

## 功能描述

修复 API 测试套件中的 26 个失败测试，解决数据库连接错误和测试框架冲突问题，使测试通过率达到 100%。

## 背景

API 测试套件当前有 26/30 测试失败，通过率仅 13.3%。主要问题是数据库连接错误和测试框架冲突。

## 问题分析

### 当前状态
- 测试文件总数: 4
- 通过: 4 个测试
- 失败: 26 个测试
- **通过率: 13.3%** ❌ (目标 >95%)

### 失败详情

#### 1. 数据库连接错误（主要问题）
**错误**: `SASL: SCRAM-SERVER-FIRST-MESSAGE: client password must be a string`

**影响测试文件**:
- `tests/works.test.ts` - 11 个测试失败
- `tests/fields.test.ts` - 8 个测试失败
- `tests/publish.test.ts` - 7 个测试失败

**根本原因**:
- `src/db/connection.ts` 使用环境变量 `DATABASE_*`
- 但 `.env` 文件使用 `POSTGRES_*` 变量名
- 测试运行时密码为 undefined，导致连接失败

#### 2. 测试框架冲突
**文件**: `src/services/__tests__/ai-video.service.test.ts`
**错误**: `Vitest cannot be imported in a CommonJS module using require()`

**根本原因**:
- 该文件使用 vitest 框架
- 项目配置使用 jest (jest.config.js)
- 需要统一为 jest

## 解决方案

### 方案 1: 环境配置 ✅
创建 `.env.test` 文件，使用正确的环境变量名：

```env
NODE_ENV=test
DATABASE_HOST=localhost
DATABASE_PORT=5432
DATABASE_NAME=timescaledb
DATABASE_USER=cecelia
DATABASE_PASSWORD=cecelia
```

### 方案 2: 修复 ai-video.service.test.ts
将 vitest 语法转换为 jest：

**需要修改**:
1. 删除 `import { describe, it, expect, vi, beforeEach } from 'vitest'`
2. 替换 `vi.mock` → `jest.mock`
3. 替换 `vi.clearAllMocks()` → `jest.clearAllMocks()`
4. 替换所有 `vi.fn()` → `jest.fn()`
5. 替换类型 `ReturnType<typeof vi.fn>` → `jest.MockedFunction<typeof pool.query>`

**共计 8 处改动**。

### 方案 3: Jest 配置（如需要）
确保 jest 在测试前加载 `.env.test`:

```javascript
// jest.config.js
module.exports = {
  // ... existing config
  setupFiles: ['dotenv/config'],
  testEnvironment: 'node',
  roots: ['<rootDir>/src', '<rootDir>/tests'],
};
```
