## PRD - 重构工作台为钻取式 Dashboard

**需求来源**: 用户反馈 - 现有页面平铺、没有层级关系、不知道从哪看起

**功能描述**: 
将现有的 Dashboard.tsx 重构为真正的 Executive Dashboard（总览页面），支持钻取（Drill Down）到详细页面。

**核心改动**:

1. **保留现有功能**（不删除）:
   - 欢迎语、动态问候
   - 天气、倒计时、每日一言
   - 4个统计卡片（今日发布、待处理任务、活跃账号、AI 执行）
   - 最近活动列表

2. **重构业务模块区域**（关键改动）:
   - 移除现有的"快捷操作"和"数据采集面板"
   - 新增：4个可点击的业务模块卡片（替换现有的 MAIN_FEATURES）
   
   **新业务模块**:
   ```
   ┌──────────────┬──────────────┐
   │ 新媒体运营   │ AI 能力中心  │
   │ 7平台运营中  │ 8个AI员工    │
   │ 今日发布3条  │ 视频生成3次  │
   │ [查看详情→] │ [查看详情→]  │
   ├──────────────┼──────────────┤
   │ 数据中心     │ 系统监控     │
   │ 播放1.2M     │ 服务正常     │
   │ 互动率5%     │ 5任务运行中  │
   │ [查看详情→] │ [查看详情→]  │
   └──────────────┴──────────────┘
   ```

3. **钻取路径**:
   - 新媒体运营 → 跳转到 /media
   - AI 能力中心 → 跳转到 /ai
   - 数据中心 → 跳转到 /data  
   - 系统监控 → 跳转到 /system

4. **数据获取**:
   - 从现有 API 获取汇总数据
   - `/api/v1/dashboard/stats` - 已有接口
   - 新增字段（如果需要）：平台数量、AI员工数量等

**涉及文件**:
- `apps/dashboard/src/pages/Dashboard.tsx` - 主要改动
- `apps/dashboard/src/api/index.ts` - API 接口（可能需要补充）
- `apps/dashboard/src/config/navigation.config.ts` - 新增路由（/ai, /data, /system）

**成功标准**:
1. ✅ 工作台显示 4 个业务模块卡片
2. ✅ 每个卡片显示汇总数据（平台数量、今日发布、AI员工数等）
3. ✅ 点击卡片能跳转到对应的详情页面（路由正确）
4. ✅ 保持现有的欢迎语、统计卡片、最近活动功能
5. ✅ 样式美观，符合现有设计风格
6. ✅ 响应式布局（桌面2x2、平板1x2、手机堆叠）

**非目标**（明确不做）:
- ❌ 不创建 /media、/ai、/data、/system 的详情页面（仅做跳转）
- ❌ 不修改其他现有页面（ExecutionStatus、PlatformDataPage等）
- ❌ 不做 WebSocket 实时更新（保留在后续任务）
- ❌ 不做大规模的导航重构（仅在 Dashboard 页面改动）

**备注**:
这是第一阶段（Phase 1），只做工作台重构。后续会有：
- Phase 2: 创建二级页面（/media、/ai等）
- Phase 3: 添加系统监控（WebSocket）
