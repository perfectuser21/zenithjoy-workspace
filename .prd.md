## PRD - 平台数据展示页面

**需求来源**: "在 Dashboard 添加平台数据展示页面，展示所有采集的原始数据"

**功能描述**:
在 ZenithJoy Dashboard 中添加一个新页面，用于展示所有 7 个社交媒体平台采集的原始数据。数据来自 PostgreSQL TimescaleDB，包含 douyin、kuaishou、xiaohongshu、toutiao、weibo、zhihu、channels 平台的内容快照。

**核心要求**:
1. 展示每个平台的原始字段（不是统一 schema）
   - douyin: aweme_id, title, views, likes, comments, shares, favorites, publishTime
   - weibo: weibo_id, content, views, likes, comments, reposts, publishTime
   - xiaohongshu: note_id, title, views, likes, comments, shares, favorites, exposure, publishTime
   - kuaishou: photo_id, caption, views, likes, comments, shares, favorites, publishTime
   - toutiao: article_id, title, views, likes, comments, shares, favorites, publishTime
   - zhihu: question_id, title, views, voteup_count, comments, favorites, publishTime
   - channels: title, views, likes, comments, shares, favorites, publishTime

2. 连接香港生产数据库（通过 SSH tunnel: localhost:5433 → hk:5432）

3. 平台切换和筛选功能

4. 类似 Excel 的表格展示形式

**涉及文件**:
- `apps/dashboard/src/pages/PlatformDataPage.tsx` - 新页面组件
- `apps/dashboard/src/api/platform-data.api.ts` - API 接口
- `apps/dashboard/src/routes/index.tsx` - 添加路由
- Backend API endpoint for database connection

**技术方案**:
- 前端: React + TypeScript + 现有 UI 库
- 数据源: PostgreSQL TimescaleDB (raw_data views)
- 连接方式: 通过 backend API 连接数据库

**成功标准**:
1. ✅ 页面能正确连接到香港数据库并获取数据
2. ✅ 展示所有 7 个平台的数据
3. ✅ 每个平台显示其特有的原始字段
4. ✅ 数据以表格形式展示（类似 Excel）
5. ✅ 能够按平台筛选和切换
6. ✅ 在导航中可访问新页面

**非目标**:
- ❌ 不统一字段名（保留平台原始字段）
- ❌ 不做数据分析或图表展示（只展示原始数据）
- ❌ 不使用第三方数据库工具（pgweb、Adminer 等）
