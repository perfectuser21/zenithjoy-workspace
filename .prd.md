# PRD: 质量体系紧急止血

## 问题
前端代码质量门禁过弱：
1. CI 允许 100 个 lint warnings（代码持续变乱）
2. 无安全漏洞检查（npm audit）
3. 无测试框架（无法写测试）
4. 无功能清单（不知道哪些功能已跑通）

## 根本原因
质量体系缺失，导致：
- 代码越改越乱
- 改 A 坏 B 无法检测
- 无法保护已完成的功能

## 解决方案（止血三步）

### 1. 修 CI 配置
- Lint warnings: 100 → 0（阻止继续变乱）
- 添加 security job（npm audit）

### 2. 安装测试框架
- vitest + @testing-library/react
- 配置 vitest.config.ts
- 添加测试脚本（但先不写测试）

### 3. 梳理功能清单
- 创建 FEATURES_STATUS.md
- 标记每个功能状态（✅ / 🟡 / ❌）
- 为后续"边做边测"准备

## 影响范围
- `.github/workflows/ci.yml` - CI 配置
- `apps/dashboard/package.json` - 添加测试依赖
- `apps/dashboard/vitest.config.ts` - 新建配置
- `FEATURES_STATUS.md` - 新建功能清单

## 验收标准
1. CI lint 允许 0 warnings（不是 100）
2. CI 有 security job（npm audit）
3. vitest 已安装并配置
4. FEATURES_STATUS.md 已创建
5. 所有 PR 必须通过新的 CI 检查
