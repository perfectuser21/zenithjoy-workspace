# Task Card: WorkDetailPage 全链路 Tab

## 目标
在 WorkDetailPage 添加"全链路"Tab，让用户在前端看到每个作品的完整生命周期：
生成 → 发布 → 数据采集 → 日报汇总

## 背景
- `zenithjoy.works` 和 `zenithjoy.publish_logs` 已有真实数据（8 works, 47 publish_logs）
- 现有 WorkDetailPage.tsx 只有编辑功能，无法查看发布状态和数据
- 用户需要在一个页面看到内容全链路状态

## 实现范围
1. WorkDetailPage.tsx — 添加 Tab 切换 (编辑 | 全链路)
2. 全链路 Tab 展示 4 个阶段：
   - 阶段一：内容生成（从 work 数据展示）
   - 阶段二：平台发布（调用 getPublishLogs API）
   - 阶段三：数据采集（从 publish_logs.response 提取指标）
   - 阶段四：日报汇总（占位，说明由系统汇总）

## DoD
- [ ] WorkDetailPage 顶部有 编辑/全链路 Tab 切换
- [ ] 全链路 Tab 有 4 个阶段卡片（进度概览 + 详情区域）
- [ ] 阶段二展示每个平台的发布状态（颜色区分 成功/失败/进行中/待发布）
- [ ] 阶段三：有指标时展示数字，无指标时显示"待采集"状态
- [ ] 发布中每 30 秒自动刷新发布日志
- [ ] TypeScript 无类型错误
- [ ] 仅修改 WorkDetailPage.tsx，无额外文件
