# DoD - Works List Frontend (Database View)

## 验收标准

### 功能验收

#### 1. 作品列表显示
- [ ] 打开 /works 页面，显示所有作品（标题、类型、状态、日期等字段）
      Test: manual:访问 http://localhost:5173/works 截图验证
- [ ] 空状态时显示 "暂无作品" 提示
      Test: manual:清空数据库后访问页面验证

#### 2. 筛选功能
- [ ] 点击类型筛选，只显示对应类型的作品
      Test: manual:选择"图文"类型，验证只显示图文作品
- [ ] 点击状态筛选，只显示对应状态的作品
      Test: manual:选择"已发布"状态，验证只显示已发布作品
- [ ] 筛选器可组合使用（type + status）
      Test: manual:同时选择"图文"+"已发布"，验证结果正确

#### 3. 排序功能
- [ ] 点击表头，按该字段排序
      Test: manual:点击"创建日期"表头，验证排序
- [ ] 再次点击，切换升序/降序
      Test: manual:再次点击"创建日期"，验证排序方向切换
- [ ] 显示排序方向指示（↑↓ 图标）
      Test: manual:排序时截图验证图标显示

#### 4. 搜索功能
- [ ] 输入关键词，实时或按回车搜索
      Test: manual:输入"AI"，验证搜索结果
- [ ] 搜索匹配标题和内容
      Test: manual:输入标题关键词和正文关键词，验证都能搜到
- [ ] 清空搜索框，恢复所有数据
      Test: manual:清空搜索框，验证显示所有作品

#### 5. 分页功能
- [ ] 显示正确的总数和当前页
      Test: manual:底部显示"显示 1-20 / 共 X 条"
- [ ] 点击下一页，加载下一页数据
      Test: manual:点击下一页，验证显示第 21-40 条
- [ ] 切换每页显示数量（10/20/50/100）
      Test: manual:切换每页 50 条，验证显示正确
- [ ] 分页时保持筛选和排序状态
      Test: manual:筛选后翻页，验证筛选条件保持

#### 6. 新建作品
- [ ] 点击 "+ 新建" 按钮，打开对话框
      Test: manual:点击按钮，验证对话框打开
- [ ] 填写标题和类型，提交成功
      Test: manual:填写"测试作品"+"图文"，点击提交
- [ ] 创建成功后，列表刷新显示新作品
      Test: manual:提交后验证列表中出现新作品
- [ ] 验证必填字段（标题、类型）
      Test: manual:留空标题提交，验证提示错误

### UI/UX 验收

#### 7. 响应式设计
- [ ] 桌面端（1920x1080）正常显示
      Test: manual:全屏模式截图验证
- [ ] 笔记本（1366x768）正常显示
      Test: manual:缩小窗口到 1366x768 验证
- [ ] 平板（768px）可用
      Test: manual:缩小窗口到 768px 验证

#### 8. 加载状态
- [ ] 数据加载时显示 Loading 状态
      Test: manual:刷新页面时截图验证 Loading 状态
- [ ] 加载失败时显示错误提示
      Test: manual:停止 API 服务，验证错误提示

#### 9. 交互反馈
- [ ] 点击行时有 hover 效果
      Test: manual:鼠标悬停在行上，截图验证 hover 效果
- [ ] 按钮有 hover 和 active 状态
      Test: manual:鼠标悬停按钮，截图验证状态
- [ ] 表单验证错误有提示
      Test: manual:提交空表单，验证错误提示

### 代码质量验收

#### 10. TypeScript 类型
- [ ] 所有组件有完整类型定义
      Test: manual:运行 npx tsc --noEmit 验证无类型错误
- [ ] API 调用有类型推断
      Test: manual:检查 api/works.ts 文件，验证类型定义

#### 11. 错误处理
- [ ] API 失败时显示错误提示
      Test: manual:停止 API 服务，验证错误提示
- [ ] 网络错误时有重试机制
      Test: manual:断网后重连，验证 React Query 自动重试

### 集成验收

#### 12. API 集成
- [ ] 成功调用 GET /api/works 获取列表
      Test: manual:打开 DevTools Network，验证 API 调用
- [ ] 成功调用 POST /api/works 创建作品
      Test: manual:创建作品时验证 API 调用
- [ ] 筛选参数正确传递给 API
      Test: manual:筛选时验证 URL 参数（type, status, search 等）
- [ ] 分页参数正确传递给 API
      Test: manual:翻页时验证 URL 参数（limit, offset）

### 测试验收

- [ ] 前端构建成功（npm run build:dashboard）
      Test: manual:运行构建命令，验证无错误
- [ ] 开发服务器正常启动（npm run dev:dashboard）
      Test: manual:启动开发服务器，访问 http://localhost:5173/works
