# PRD: 今日头条微头条自动发布功能

## 1. 背景与目标

### 背景
- 已完成文章发布功能（长图文，有标题）✅
- 用户要求重点实现微头条发布功能
- 微头条是今日头条的短内容形式，类似微博

### 目标
实现微头条自动发布功能，通过 CDP 连接 Windows 浏览器自动化完成发布流程。

## 2. 需求分析

### 2.1 微头条特点

根据调查和用户描述：
- **内容形式**：图片 + 内容 + 标题（待验证UI确认）
- **可选功能**：话题标签、位置信息
- **发布入口**：`https://mp.toutiao.com/profile_v4/weitoutiao/publish`（推测）

### 2.2 与文章发布的对比

| 项目 | 文章 | 微头条 |
|------|------|--------|
| 标题 | 必填（2-30字） | 待确认（用户说有） |
| 内容 | 长文本 | 短文本 |
| 图片 | 1-9张 | 待确认 |
| 上传方式 | Drawer | 待探索 |
| 话题标签 | 无 | 支持 |
| 位置信息 | 无 | 支持 |

## 3. 功能需求

### 3.1 核心功能

#### F1: UI探索（第一步，必须先完成）
- **输入**：无
- **输出**：微头条发布页面的完整UI结构文档
- **步骤**：
  1. 导航到微头条发布页面
  2. 截图并分析页面结构
  3. 记录：
     - 标题输入框的选择器（如果有）
     - 内容输入框的选择器
     - 图片上传按钮和file input位置
     - 话题标签添加方式
     - 位置信息添加方式
     - 发布按钮位置
  4. 生成UI探索报告：`/tmp/weitoutiao-ui-report.md`

#### F2: 内容填写
- **输入**：标题（如果有）、正文内容
- **处理**：
  - 使用 React 受控输入方式（参考文章发布）
  - 标题：`Object.getOwnPropertyDescriptor` + `Input.insertText`
  - 内容：可能是普通 textarea 或 ProseMirror 编辑器
- **输出**：标题和内容已填写

#### F3: 图片上传
- **输入**：图片文件路径数组（Windows本地路径）
- **处理**：
  - 探索上传方式（可能也是drawer方式，参考文章发布）
  - 找到 file input 元素
  - 使用 `DOM.setFileInputFiles` 设置文件
  - 触发 change 事件
- **输出**：图片上传成功

#### F4: 可选功能配置
- **输入**：话题标签、位置信息（可选）
- **处理**：
  - 话题标签：可能直接在内容中输入 `#话题#`
  - 位置信息：探索位置添加方式
- **输出**：可选功能已配置

#### F5: 发布
- **输入**：无
- **处理**：
  1. 点击"发布"按钮
  2. 处理确认弹窗（如果有）
  3. 等待页面跳转
  4. 验证发布成功
- **输出**：发布成功，返回发布URL

### 3.2 错误处理

- **标题超长**：自动截断到最大长度
- **内容为空**：报错并跳过
- **图片上传失败**：重试3次，失败后报错
- **发布失败**：截图保存，报错退出

### 3.3 日志与调试

- **截图保存**：每个关键步骤保存截图到 `/tmp/weitoutiao-screenshots/`
- **日志输出**：清晰的步骤说明和状态输出
- **错误截图**：失败时保存错误截图

### 4.3 参考已完成的文章发布

**关键技术点**（从文章发布复用）：

1. **React 受控输入**
```javascript
const setter = Object.getOwnPropertyDescriptor(
  Object.getPrototypeOf(textarea), 'value'
).set;
setter.call(textarea, value);
textarea.dispatchEvent(new Event('input', { bubbles: true }));
```

2. **Drawer 文件上传**
```javascript
// 1. 点击打开drawer
document.querySelector('.article-cover-add').click(); // 可能不同

// 2. 等待drawer出现
await sleep(2000);

// 3. 找到file input
const { nodeIds } = await cdp.send('DOM.querySelectorAll', {
  nodeId: root.nodeId,
  selector: 'input[type="file"]'
});

// 4. 设置文件
await cdp.send('DOM.setFileInputFiles', {
  nodeId: nodeIds[0],
  files: images
});
```

3. **发布确认**
```javascript
// 点击发布按钮
const publishBtn = buttons.find(b =>
  b.textContent.includes('发布') &&
  b.offsetWidth > 0
);
publishBtn.click();

// 处理确认弹窗
if (confirmModal) {
  const confirmBtn = buttons.find(b => b.textContent.includes('确认'));
  confirmBtn.click();
}
```

## 8. 风险与注意事项

### 8.1 待确认的假设
- ⚠️ 微头条是否真的有标题字段？（用户说有，但需UI确认）
- ⚠️ 图片上传方式是否与文章相同？

- CDP连接可能不稳定 → 添加重试机制
- 元素选择器可能变化 → 使用多个备选选择器
- 验证码可能出现 → 需要手动处理

### 8.3 兼容性
- 必须使用Windows本地路径（`C:\...`）
- 图片格式：JPG, PNG
- 图片大小：建议不超过5MB
