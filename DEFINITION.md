# ZenithJoy System Definition

## 概述

ZenithJoy 是一个综合性的业务管理和自动化平台，旨在通过 AI 技术和自动化流程提升企业运营效率。

## 系统架构

### 核心组件

#### 1. Workspace（工作空间）
- **定位**: 统一的前端管理界面
- **技术栈**: React + TypeScript + Vite
- **端口**: 5213（开发）/ 5214（生产）
- **职责**:
  - 项目管理
  - 任务调度
  - 数据可视化
  - 系统监控

#### 2. Creator（内容创作系统）
- **定位**: AI 驱动的内容生成和管理平台
- **技术栈**: Next.js + Python Backend
- **职责**:
  - 文章生成
  - 图片创作
  - 视频制作
  - 内容发布

#### 3. GeoAI（地理 AI 分析）
- **定位**: 地理数据智能分析系统
- **技术栈**: Python + PostGIS + Leaflet
- **职责**:
  - 地理数据处理
  - 空间分析
  - 位置智能
  - 地图可视化

#### 4. JNSY-Label（标签管理系统）
- **定位**: 统一的标签和分类管理平台
- **技术栈**: Node.js + PostgreSQL
- **职责**:
  - 标签体系管理
  - 内容分类
  - 标签智能推荐
  - 数据标注

#### 5. Workflows（工作流引擎）
- **定位**: 业务流程自动化平台
- **技术栈**: n8n + Custom Workflows
- **职责**:
  - 流程自动化
  - 任务编排
  - 事件触发
  - 集成管理

### 数据库架构

- **主数据库**: PostgreSQL (zenithjoy)
  - 业务数据
  - 用户管理
  - 系统配置

- **缓存层**: Redis
  - Session 管理
  - 热点数据缓存
  - 任务队列

- **文件存储**: NAS
  - 媒体文件
  - 文档存储
  - 备份数据

## 开发规范

### 代码规范
- TypeScript for 类型安全
- ESLint + Prettier for 代码格式
- Conventional Commits for 提交信息
- GitHub Flow for 分支管理

### 测试策略
- 单元测试: Jest
- 集成测试: Supertest
- E2E 测试: Playwright
- 测试覆盖率: > 80%

### CI/CD 流程
1. 代码提交触发 GitHub Actions
2. 运行测试和代码质量检查
3. 构建 Docker 镜像
4. 部署到相应环境
5. 运行健康检查

## 安全策略

### 认证授权
- JWT Token 认证
- RBAC 权限控制
- API Rate Limiting
- IP 白名单（可选）

### 数据安全
- 数据库加密存储
- HTTPS 传输加密
- 敏感信息脱敏
- 定期安全审计

### 备份策略
- 数据库每日备份
- 代码仓库多地备份
- 配置文件加密备份
- 灾难恢复计划
