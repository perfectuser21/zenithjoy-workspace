# PRD: 修复 ToAPI 大写状态映射问题

## 问题描述

用户反馈：AI 视频生成任务最终显示 "Task timeout"，任务实际已完成但前端一直显示"生成中"，直到 5 分钟超时。

## 根本原因

通过浏览器 console 日志发现：

```javascript
status: "SUCCESS"  // ToAPI 返回大写
fail_reason: "https://files.toapis.com/flow/713b2892-f218-45a9-9046-53c3394e52f2.mp4"
```

**问题**：
1. ToAPI API 返回的状态值是大写：`"SUCCESS"`, `"FAILED"`, `"PROCESSING"`
2. 当前代码的 switch 语句只匹配小写：`case 'success'`, `case 'failed'`, `case 'processing'`
3. 大写状态不匹配任何 case，导致 status 保持默认值 `'queued'`
4. 任务永远不会完成，轮询持续 5 分钟直到超时

## 解决方案

在 `apps/dashboard/src/api/platforms/toapi.ts` 的 `mapToUnifiedTask()` 方法中：

**方案 1**（推荐）：统一转换为小写
```typescript
switch (task.status.toLowerCase()) {
  case 'queued': ...
  case 'success': ...
  case 'failed': ...
}
```

**方案 2**：添加大写 case
```typescript
switch (task.status) {
  case 'SUCCESS':
  case 'success':
    status = 'completed';
    break;
  case 'FAILED':
  case 'failed':
    status = 'failed';
    break;
}
```

推荐使用方案 1，因为更健壮，可以处理任何大小写组合。

## 影响范围

- 文件：`apps/dashboard/src/api/platforms/toapi.ts`
- 影响：所有 ToAPI 视频生成任务的状态映射
- 严重程度：**Critical** - 功能完全不可用

## 验收标准

1. 修改 `mapToUnifiedTask()` 方法，支持大写状态
2. 更新相关测试，覆盖大写和小写状态
3. 手动测试：提交视频生成任务，等待完成，验证不再超时
4. 验证任务完成后正确显示视频预览界面

## 测试场景

1. 提交视频生成任务
2. 观察 TaskMonitor 显示状态变化
3. 等待任务完成（不超时）
4. 验证显示视频预览而不是 "Task timeout" 错误
