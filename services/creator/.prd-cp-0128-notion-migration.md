# PRD: Notion 作品库迁移到本地 Markdown

## 背景

用户的所有创作内容目前存储在 Notion 的 "ZenithJoy_作品库" 数据库中。需要将 2025年11月之后的内容迁移到本地 Git 仓库，便于版本管理和自动化发布。

## 目标

将 Notion 作品库中的内容导出为 Markdown 文件，按类型组织存储。

## 范围

- 时间范围：2025年11月1日之后的内容
- 内容类型：深度帖、短贴、文章
- 不迁移短视频内容（只保留链接）

## 功能需求

### 1. 目录结构

```
content/
├── deep-posts/      # 深度帖
├── short-posts/     # 短贴
└── articles/        # 文章
```

### 2. 文件命名

格式：`YYYY-MM-DD-标题.md`

### 3. 文件格式

每个 Markdown 文件包含：
- YAML frontmatter（标题、日期、类型、主题、状态、爆款/精品标记、Notion ID）
- 正文内容（段落、标题、列表、引用、图片链接）

### 4. 导出脚本

创建 Python 脚本 `scripts/notion-export.py`：
- 调用 Notion API 获取数据库内容
- 转换 blocks 为 Markdown 格式
- 按类型保存到对应目录

## 数据统计

从 Notion 查询结果：
- 深度帖：59 条
- 短贴：40 条
- 深度文章：1 条
- 共计：100 条

## 验收标准

1. 所有 100 条内容成功导出为 Markdown 文件
2. 文件按类型正确分类到对应目录
3. frontmatter 包含必要的元数据
4. 正文内容完整保留
