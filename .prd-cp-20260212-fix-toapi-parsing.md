# PRD - 修复 ToAPI 响应解析 - 处理嵌套的 data 结构和 SUCCESS 状态映射

## 功能描述

修复 AI 视频生成功能中 ToAPI 响应解析的 bug，使视频生成完成后能正确显示"已完成"状态，而不是错误地停留在"排队中"状态。

## 成功标准

- 视频生成完成后，状态立即从"排队中"变为"已完成"
- 进度条正确显示 100% 而不是 0%
- 视频 URL 正确提取并可播放
- 状态轮询在视频完成后停止

## 背景

用户反馈视频生成功能存在严重问题：
1. ~~点击"开始生成视频"按钮后页面空白（已在 PR #43 修复）~~
2. **视频显示为"排队中"状态长达 3 分 46 秒，实际上视频已经生成完成**

通过用户提供的 Console 日志分析，发现 ToAPI 返回的响应结构与我们的解析代码不匹配。

## 问题分析

### ToAPI 实际响应结构

```json
{
  "code": "success",
  "data": {
    "task_id": "task_01KH8CVNNMPJZ546CACNYFWNJC",
    "status": "SUCCESS",
    "progress": "100%",
    "fail_reason": "https://upload.apimart.ai/f/video/9998229117699660-d1fc84f1-660b-4d74-a7ec-6c970985a0a2-video_task_01KH8CVNNMPJZ546CACNYFWNJC.mp4",
    "data": {
      "id": "task_01KH8CVNNMPJZ546CACNYFWNJC",
      "status": "queued",
      "progress": 0,
      "created_at": 1739326797,
      "object": "generation.task"
    }
  }
}
```

### 当前代码的错误

**文件**: `apps/dashboard/src/api/video-generation.api.ts`

1. **嵌套结构未解包** (lines 30-48, 95-110)
   - `createVideoGeneration`: 直接返回 `response.data`，但 ToAPI 返回 `{code, data}` 结构
   - `getTaskStatus`: 同样问题，未解包外层的 `{code, data}`
   - 结果：`mapToAPIResponse` 收到的是 `{code, data}` 整体，不是 `data` 内容

2. **错误的字段路径** (lines 55-90)
   - 我们读取 `rawResponse.status`，但实际应该读 `rawResponse.data.status`（因为解包不完整）
   - 当前读到的是 `data.data.status` (queued)，而不是 `data.status` (SUCCESS)

3. **大写状态值未处理** (lines 59-69)
   - ToAPI 返回大写: "SUCCESS", "FAILED", "PROCESSING"
   - 我们的 statusMap 只有小写 key
   - 虽然有 `.toLowerCase()` (line 71)，但因为读取路径错误，实际读到的是内层的小写 "queued"

4. **进度格式不匹配** (line 73)
   - ToAPI 返回字符串 "100%"
   - 我们期望数字 100
   - 需要解析: `parseInt(progress.replace('%', ''))`

5. **视频 URL 字段错误** (line 77)
   - ToAPI 把视频 URL 放在 `fail_reason` 字段（这是 ToAPI 的 API bug）
   - 我们期望在 `result.video_url`
   - 需要映射: `result: { video_url: rawResponse.fail_reason || rawResponse.result?.video_url }`

## 解决方案

### 1. 修复响应解包

在 `createVideoGeneration` 和 `getTaskStatus` 中，正确解包 ToAPI 的 `{code, data}` 结构：

```typescript
// Before
const response = await axios.post(url, payload, { headers });
return mapToAPIResponse(response.data);

// After
const response = await axios.post(url, payload, { headers });
const toApiResponse = response.data;
if (toApiResponse.code === 'success' && toApiResponse.data) {
  return mapToAPIResponse(toApiResponse.data);  // 只传 data 部分
}
throw new Error(`ToAPI error: ${toApiResponse.code}`);
```

### 2. 更新 mapToAPIResponse

```typescript
export function mapToAPIResponse(rawResponse: any): GetTaskResponse {
  console.log('[ToAPI Debug] Raw response (after unwrap):', JSON.stringify(rawResponse, null, 2));

  // ToAPI 返回大写状态值: SUCCESS, FAILED, PROCESSING
  const status = rawResponse.status;

  const statusMap: Record<string, TaskStatus> = {
    'SUCCESS': 'completed',
    'FAILED': 'failed',
    'PROCESSING': 'in_progress',
    'pending': 'queued',
    'queued': 'queued',
    'processing': 'in_progress',
    'in_progress': 'in_progress',
    'completed': 'completed',
    'success': 'completed',
    'failed': 'failed',
    'error': 'failed',
  };

  const mappedStatus: TaskStatus = statusMap[status] || 'queued';

  // 解析进度: "100%" -> 100
  let progress = 0;
  if (typeof rawResponse.progress === 'string') {
    progress = parseInt(rawResponse.progress.replace('%', '')) || 0;
  } else {
    progress = rawResponse.progress || 0;
  }

  // ToAPI bug: 视频 URL 在 fail_reason 字段
  let result = rawResponse.result;
  if (rawResponse.fail_reason && rawResponse.fail_reason.startsWith('http')) {
    result = { video_url: rawResponse.fail_reason };
  }

  return {
    id: rawResponse.task_id || rawResponse.id,
    object: rawResponse.object || 'generation.task',
    status: mappedStatus,
    progress,
    created_at: rawResponse.created_at,
    completed_at: rawResponse.completed_at,
    error: rawResponse.error,
    result,
  };
}
```
