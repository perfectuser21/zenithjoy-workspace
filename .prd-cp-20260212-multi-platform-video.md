# PRD - AI 视频生成多平台支持架构

## 功能描述

重构 AI 视频生成功能，支持多个视频生成平台，每个平台有独立的配置和参数。

## 成功标准

- 用户可以选择不同的平台（ToAPI, Replicate 等）
- 每个平台显示正确的模型列表和参数选项
- 统一的 API 抽象层，支持添加新平台
- 正确处理平台特定的响应格式

## 背景

当前问题：
1. 代码中的模型名（MiniMax-Hailuo-02, Sora-2 等）与 ToAPI 实际支持的模型（veo3.1-fast, veo3.1-quality）完全不匹配
2. 参数格式不对：我们发送 `resolution: "768p"`, ToAPI 要求 `aspect_ratio: "16:9"`
3. 图片格式不对：我们可能发送 base64, ToAPI 要求 URL
4. 时长不对：我们支持 5/10 秒，ToAPI 固定 8 秒

根本原因：代码写的是假想的统一 API，但实际上每个平台的 API 都不一样。

## 解决方案

### 1. 架构设计

```typescript
// 平台抽象层
interface VideoPlatform {
  id: string;
  name: string;
  models: ModelConfig[];
  createVideo(params: UnifiedParams): Promise<Task>;
  getTaskStatus(taskId: string): Promise<Task>;
  mapParams(params: UnifiedParams): PlatformSpecificParams;
  mapResponse(response: any): UnifiedTask;
}

// 统一参数格式（前端 UI）
interface UnifiedParams {
  platform: 'toapi' | 'replicate' | 'runwayml';
  model: string;
  prompt: string;
  duration?: number;
  aspectRatio?: '16:9' | '9:16' | '1:1';
  resolution?: '720p' | '1080p' | '4k';
  imageUrls?: string[];
}

// 统一任务格式（前端显示）
interface UnifiedTask {
  id: string;
  platform: string;
  status: 'queued' | 'in_progress' | 'completed' | 'failed';
  progress: number;
  videoUrl?: string;
  error?: string;
}
```

### 2. ToAPI 平台实现

```typescript
class ToAPIPlatform implements VideoPlatform {
  id = 'toapi';
  name = 'ToAPI';

  models = [
    {
      id: 'veo3.1-fast',
      name: 'VEO 3.1 Fast',
      description: 'Google VEO3 快速模式',
      duration: 8, // 固定
      aspectRatios: ['16:9', '9:16'],
      supportImages: true,
    },
    {
      id: 'veo3.1-quality',
      name: 'VEO 3.1 Quality',
      description: 'Google VEO3 高质量模式',
      duration: 8,
      aspectRatios: ['16:9', '9:16'],
      supportImages: true,
    }
  ];

  mapParams(params: UnifiedParams) {
    // 转换为 ToAPI 格式
    return {
      model: params.model, // veo3.1-fast
      prompt: params.prompt,
      duration: 8, // 固定
      aspect_ratio: params.aspectRatio || '16:9',
      image_urls: params.imageUrls,
      metadata: {
        generation_type: params.imageUrls?.length === 2 ? 'frame' : undefined,
        resolution: params.resolution,
      }
    };
  }

  async createVideo(params: UnifiedParams) {
    const toApiParams = this.mapParams(params);
    const response = await fetch('https://toapis.com/v1/videos/generations', {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${API_KEY}`,
        'Content-Type': 'application/json',
      },
      body: JSON.stringify(toApiParams)
    });

    const data = await response.json();
    return this.mapResponse(data);
  }

  mapResponse(data: any): UnifiedTask {
    return {
      id: data.id,
      platform: 'toapi',
      status: data.status === 'queued' ? 'queued' :
              data.status === 'in_progress' ? 'in_progress' :
              data.status === 'completed' ? 'completed' : 'failed',
      progress: data.progress || 0,
      videoUrl: data.result?.video_url,
      error: data.error?.message,
    };
  }
}
```

## ToAPI 平台详细规格

根据官方文档 https://docs.toapis.com/docs/cn/api-reference/videos/veo3/generation：

### 请求格式
```json
{
  "model": "veo3.1-fast" | "veo3.1-quality",
  "prompt": "视频描述",
  "duration": 8,  // 固定
  "aspect_ratio": "16:9" | "9:16",
  "image_urls": ["url1", "url2"],  // 可选，仅支持 URL
  "metadata": {
    "generation_type": "frame" | "reference",
    "resolution": "720p" | "1080p" | "4k",
    "enable_gif": false
  }
}
```

### 响应格式
```json
{
  "id": "video_xxx",
  "object": "generation.task",
  "model": "veo3.1-fast",
  "status": "queued" | "in_progress" | "completed" | "failed",
  "progress": 0-100,
  "created_at": 1768380224,
  "metadata": {"aspect_ratio": "16:9"}
}
```

### 状态查询
```
GET https://toapis.com/v1/videos/generations/{id}
Authorization: Bearer {API_KEY}
```

### 重要限制
- 视频时长固定 8 秒
- 图片必须是 URL，不支持 base64
- 视频链接有效期 24 小时

## 技术要点

1. **平台注册机制**：使用工厂模式，方便添加新平台
2. **参数映射**：统一 → 平台特定 → 统一（双向映射）
3. **错误处理**：平台特定错误 → 统一错误格式
4. **类型安全**：TypeScript 严格类型检查
5. **向后兼容**：旧的 localStorage 数据迁移

## 非功能需求

- 平台切换无刷新
- 参数验证在客户端完成
- 错误信息清晰易懂
- 支持任务历史记录（跨平台）
- API Key 按平台独立配置

## 风险和缓解

| 风险 | 缓解措施 |
|------|----------|
| 平台 API 变化 | 版本化适配器，保留旧版本 |
| 平台不可用 | 显示状态，支持平台禁用 |
| 参数不兼容 | UI 根据平台动态调整 |
| 响应格式差异 | 严格的类型映射和测试 |
