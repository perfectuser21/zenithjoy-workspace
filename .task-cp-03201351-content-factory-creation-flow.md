# Task Card: 内容工厂创作链路 — 执行按钮 + Codex 派发 + 产出预览

## 需求（What & Why）
**功能描述**:
1. 前端 ContentFactoryPage 加「开始执行」按钮，创建后点击即触发 pipeline 执行
2. 前端加目标平台勾选（抖音/小红书/公众号/知乎/微博/头条/快手/视频号）
3. 后端加 `POST /api/brain/pipelines/:id/run` API，直接调用 executor 不依赖 tick
4. executor 改成通过 codex-bridge 派发到西安 Mac mini 执行
5. task-router content-* 路由从 `us` 改成 `xian`
6. 前端产出预览：Pipeline 完成后展示卡片图片 + 文案
7. 进度条自动轮询（每 5 秒刷新状态）

**背景**: 内容工厂 pipeline 代码全部写好了（orchestrator + executor + 前端），但：
- 没有手动触发机制（tick 调度器停了）
- 前端缺少执行按钮和产出预览
- executor 在本地跑，需要改成派发到西安 Codex
- 已验证西安 Codex + card-renderer.mjs 模板可以生成与 Claude Code 一致质量的卡片

**不做什么**:
- 不做发布功能（创作和发布分开，发布是后续任务）
- 不改 orchestrator 状态机逻辑（已经写好了）
- 不改 card-renderer.mjs 模板（已验证通过）

## 成功标准
1. [BEHAVIOR] 前端创建 Pipeline 后，出现「开始执行」按钮，点击后 pipeline 状态变为 in_progress
2. [BEHAVIOR] 执行过程中进度条自动更新（research → generate → review → export）
3. [BEHAVIOR] Pipeline 完成后，前端展示生成的卡片图片预览
4. [ARTIFACT] 后端 `POST /api/brain/pipelines/:id/run` API 可用
5. [BEHAVIOR] executor 通过 codex-bridge 派发任务到西安执行
6. [GATE] 不破坏已有前端功能
