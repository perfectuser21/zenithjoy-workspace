# 安全内容指南 - 避免被平台识别为自动化

## ⚠️ CRITICAL - 内容安全规则

**绝对禁止在发布内容中包含以下字眼**，否则会被平台识别为自动化工具并封号：

### 🚫 禁止使用的词汇

| 禁止内容 | 原因 | 后果 |
|----------|------|------|
| "自动化" | 直接暴露自动化工具 | 立即封号 |
| "测试" | 明显的测试标记 | 封号 |
| "验证" | 技术测试用语 | 封号 |
| "脚本" | 技术工具标记 | 封号 |
| "机器人" / "bot" / "🤖" | 自动化标识 | 封号 |
| "API" | 技术接口标记 | 封号 |
| "CDP" / "Selenium" | 自动化工具名称 | 封号 |
| "批量" / "自动发布" | 明显的批量操作 | 封号 |

### ✅ 安全的测试内容示例

**纯文本微头条**：
```
今天天气真好啊☀️

分享一下最近读的书，收获满满！

#读书笔记 #日常分享
```

**带话题的微头条**：
```
周末去了一家新开的咖啡店 ☕

环境超级好，推荐给大家~

#咖啡 #周末时光
```

**心情分享**：
```
最近工作有点忙，但还是要保持好心情 💪

加油！

#日常 #生活感悟
```

### 🎯 内容编写原则

1. **像真人一样写**：自然、口语化、有情感
2. **不要提技术细节**：任何与开发、测试、自动化相关的词都不要出现
3. **使用emoji**：✅ 自然的emoji（😊💪☕） ❌ 机器人emoji（🤖🔧⚙️）
4. **话题标签**：使用真实的话题，不要用"#测试"、"#自动化"
5. **内容多样化**：不要发布重复或相似度极高的内容

### 📝 正确的测试配置示例

**错误示例** ❌：
```json
{
  "content": "这是一条自动化测试微头条 🤖\n\n测试内容：验证微头条自动发布功能是否正常工作。\n\n#自动化测试 #AI工具"
}
```

**正确示例** ✅：
```json
{
  "content": "今天学到一个新知识点，记录一下📝\n\n原来还可以这样理解，豁然开朗！\n\n#学习笔记 #知识分享"
}
```

### 🛡️ 防封号检查清单

在每次发布前，检查内容：

- [ ] 没有"自动化"、"测试"、"验证"等技术词汇
- [ ] 没有机器人emoji（🤖）
- [ ] 内容读起来像真人写的
- [ ] 话题标签是真实的话题
- [ ] 与上次发布的内容不重复
- [ ] 没有暴露技术细节

### 💡 推荐做法

1. **准备内容库**：提前准备一批真实的、多样化的内容
2. **随机化**：发布时间、内容、话题都要有变化
3. **模拟真人行为**：不要连续快速发布
4. **内容质量**：发布有价值的真实内容，不仅仅是测试

---

**记住**：平台的反作弊系统越来越智能，任何暴露自动化痕迹的行为都可能导致封号。

**最佳策略**：把自动化工具当作"助手"，帮你发布真实内容，而不是发布测试内容。
