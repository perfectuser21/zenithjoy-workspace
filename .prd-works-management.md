# PRD: ZenithJoy 作品管理系统

## 📋 产品愿景

**打造一个类 Notion 的内容创作与发布管理系统**，让创作者能够：
- 在前台直观地管理所有作品（文本/图文/视频/长文/音频）
- 像 Notion 一样灵活地增删自定义字段
- 一键发布到多平台（抖音、小红书、今日头条等）
- 追踪每个作品在各平台的数据表现

---

## 🎯 核心功能

### 1. 作品库（Database View）

**类似 Notion Database 的列表视图**

#### 功能点：
- ✅ 列表展示所有作品（标题、类型、发布日期、状态等）
- ✅ 多种视图：表格视图、日历视图、看板视图
- ✅ 筛选：按类型、状态、日期、标签筛选
- ✅ 排序：按任意字段排序
- ✅ 搜索：全文搜索标题和内容
- ✅ 批量操作：批量发布、归档、删除

#### UI 示例：

### 2. 作品详情（Page View）

**类似 Notion Page 的富文本编辑器，可以前台编辑**

#### 功能点：

**基础信息区**：
- ✅ 标题（可编辑）
- ✅ 内容类型（文本/图文/视频/长文/音频）
- ✅ 自定义字段（用户可增删）

**内容编辑区**（富文本）：
- ✅ 文本编辑（支持 Markdown）
- ✅ 图片上传（拖拽 / 粘贴 / 选择文件）
- ✅ 视频上传（封面 + 文件 / 外部链接）
- ✅ 双语支持（中文 / 英文切换）
- ✅ 分栏布局（图片左，文字右）
- ✅ 块元素：标题、列表、引用、代码、分割线

**媒体管理区**：
- ✅ 封面图设置
- ✅ 图片/视频列表（可排序、删除、替换）
- ✅ NAS 文件路径显示
- ✅ 预览功能

**发布管理区**：
- ✅ 平台选择（多选：抖音、小红书、今日头条等）
- ✅ 发布状态（草稿 / 待发布 / 已发布）
- ✅ 计划发布时间
- ✅ 平台链接（发布后自动填充）

**数据追踪区**：
- ✅ 各平台数据趋势图（播放量、点赞、评论等）
- ✅ 多日对比（Day 1 / Day 3 / Day 7 / Day 30）
- ✅ 关联到 TimescaleDB 的时间序列数据

### 3. 字段管理（Field Manager）

**像 Notion 一样灵活地增删字段**

#### 功能点：
- ✅ 查看所有字段（核心字段 + 自定义字段）
- ✅ 新增字段：输入字段名、选择类型（文本/单选/多选/日期/数字）
- ✅ 编辑字段：修改字段名、类型、选项
- ✅ 删除字段：隐藏字段（数据不丢失）
- ✅ 排序字段：拖拽调整显示顺序

#### 支持的字段类型：
| 类型 | 说明 | 示例 |
|------|------|------|
| 文本 | 单行文本 | "目标受众: 程序员" |
| 多行文本 | 长文本 | "内部笔记: ..." |
| 单选 | 下拉选择 | "优先级: 高/中/低" |
| 多选 | 多个标签 | "标签: [创作系统] [AI]" |
| 日期 | 日期选择器 | "截止日期: 2026-02-15" |
| 数字 | 数字输入 | "预计播放量: 100000" |
| 复选框 | 是/否 | "精品: ✓" |

## 🗄️ 数据模型

### 1. works（作品表）

```sql
CREATE TABLE zenithjoy.works (
    -- 主键
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

    -- 基础信息
    title VARCHAR(500) NOT NULL,
    body TEXT,                          -- Markdown 格式
    body_en TEXT,                       -- 英文版（可选）
    content_type VARCHAR(20) NOT NULL,  -- 'text' | 'image' | 'video' | 'article' | 'audio'

    -- 媒体文件
    cover_image VARCHAR(500),           -- 封面图 URL
    media_files JSONB,                  -- 图片/视频数组
    -- 示例：
    -- [
    --   {
    --     "type": "image",
    --     "url": "nas://media/works/2026-02/image1.jpg",
    --     "thumbnail": "nas://media/works/2026-02/image1_thumb.jpg",
    --     "size": 1024000,
    --     "caption": "图片说明",
    --     "order": 1
    --   },
    --   {
    --     "type": "video",
    --     "url": "nas://media/works/2026-02/video1.mp4",
    --     "thumbnail": "nas://media/works/2026-02/video1_thumb.jpg",
    --     "duration": 180,
    --     "size": 50000000,
    --     "order": 2
    --   }
    -- ]

    -- 平台链接（发布后）
    platform_links JSONB,
    -- 示例：
    -- {
    --   "douyin": "https://douyin.com/video/7xxx",
    --   "xiaohongshu": "https://xiaohongshu.com/xxx",
    --   "toutiao": "https://toutiao.com/xxx"
    -- }

    -- 状态
    status VARCHAR(20) DEFAULT 'draft', -- 'draft' | 'ready' | 'published' | 'archived'
    account VARCHAR(50),                -- 'XXIP' | 'XXAI'

    -- 标记
    is_featured BOOLEAN DEFAULT FALSE,  -- 精品
    is_viral BOOLEAN DEFAULT FALSE,     -- 爆款
    archived BOOLEAN DEFAULT FALSE,

    -- 时间
    created_at TIMESTAMP DEFAULT NOW(),       -- 创作日期
    scheduled_at TIMESTAMP,                   -- 计划发布
    first_published_at TIMESTAMP,             -- 首次发布
    updated_at TIMESTAMP DEFAULT NOW(),       -- 最后修改
    archived_at TIMESTAMP,                    -- 归档时间

    -- 自定义字段（用户可在前台增删）
    custom_fields JSONB
    -- 示例：
    -- {
    --   "tags": ["创作系统", "AI"],
    --   "priority": "high",
    --   "internal_notes": "这是内部笔记",
    --   "target_audience": "程序员",
    --   "mood": "开心",
    --   "任何自定义字段": "任何值"
    -- }
);

-- 索引
CREATE INDEX idx_works_status ON zenithjoy.works(status);
CREATE INDEX idx_works_type ON zenithjoy.works(content_type);
CREATE INDEX idx_works_created ON zenithjoy.works(created_at DESC);
CREATE INDEX idx_works_custom ON zenithjoy.works USING GIN (custom_fields);
```

### 2. publish_logs（发布日志表）

```sql
CREATE TABLE zenithjoy.publish_logs (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    work_id UUID REFERENCES zenithjoy.works(id) ON DELETE CASCADE,

    -- 发布信息
    platform VARCHAR(20) NOT NULL,      -- 'douyin' | 'xiaohongshu' | 'toutiao' | ...
    platform_post_id VARCHAR(100),      -- 平台返回的作品ID

    -- 状态
    status VARCHAR(20) DEFAULT 'pending', -- 'pending' | 'publishing' | 'published' | 'failed'

    -- 时间
    scheduled_at TIMESTAMP,             -- 计划发布时间
    published_at TIMESTAMP,             -- 实际发布时间
    created_at TIMESTAMP DEFAULT NOW(),

    -- 额外信息
    response JSONB,                     -- 平台返回的完整响应
    error_message TEXT,                 -- 失败原因
    retry_count INTEGER DEFAULT 0       -- 重试次数
);

-- 索引
CREATE INDEX idx_publish_work ON zenithjoy.publish_logs(work_id);
CREATE INDEX idx_publish_platform ON zenithjoy.publish_logs(platform);
CREATE INDEX idx_publish_status ON zenithjoy.publish_logs(status);
CREATE UNIQUE INDEX idx_publish_unique ON zenithjoy.publish_logs(work_id, platform);
```

### 4. platform_posts（TimescaleDB 时间序列表）

```sql
-- 扩展现有表，增加 work_id 关联
-- 在香港 TimescaleDB 执行：

-- 如果表已存在，添加字段
ALTER TABLE platform_posts ADD COLUMN IF NOT EXISTS work_id UUID;
CREATE INDEX IF NOT EXISTS idx_platform_posts_work ON platform_posts(work_id);

-- 如果表不存在，创建
CREATE TABLE IF NOT EXISTS platform_posts (
    time TIMESTAMPTZ NOT NULL,

    -- 关联
    work_id UUID,                       -- 关联到 works 表
    platform VARCHAR(20) NOT NULL,
    platform_post_id VARCHAR(100) NOT NULL,

    -- 性能指标
    play_count BIGINT DEFAULT 0,
    like_count BIGINT DEFAULT 0,
    comment_count BIGINT DEFAULT 0,
    share_count BIGINT DEFAULT 0,
    favorite_count BIGINT DEFAULT 0,
    follower_increase BIGINT DEFAULT 0,

    -- 进阶指标
    completion_rate FLOAT,              -- 完播率
    avg_play_duration FLOAT,            -- 平均播放时长
    bounce_rate_2s FLOAT,               -- 2秒跳出率

    -- 原始数据
    raw_data JSONB,

    PRIMARY KEY (time, platform, platform_post_id)
);

-- 创建 hypertable
SELECT create_hypertable('platform_posts', 'time', if_not_exists => TRUE);
```


## 📝 API 设计

### 作品管理 API

```typescript
// 获取作品列表
GET /api/works
Query:
  - status: draft | ready | published | archived
  - type: text | image | video | article | audio
  - search: string
  - sort: created_at | published_at | title
  - order: asc | desc
  - page: number
  - limit: number
Response:
  {
    items: Work[],
    total: number,
    page: number,
    limit: number
  }

// 创建作品
POST /api/works
Body:
  {
    title: string,
    body?: string,
    content_type: string,
    custom_fields?: object
  }
Response:
  { work: Work }

// 获取作品详情
GET /api/works/:id
Response:
  {
    work: Work,
    publish_logs: PublishLog[],
    analytics?: Analytics
  }

// 更新作品
PUT /api/works/:id
Body:
  {
    title?: string,
    body?: string,
    media_files?: object[],
    custom_fields?: object,
    status?: string
  }
Response:
  { work: Work }

// 删除作品
DELETE /api/works/:id
Response:
  { success: boolean }

// 上传媒体文件
POST /api/works/:id/media
Body: FormData (multipart/form-data)
  - file: File
  - type: image | video
Response:
  {
    media: {
      type: string,
      url: string,
      thumbnail: string,
      size: number
    }
  }
```

### 字段管理 API

```typescript
// 获取字段定义
GET /api/fields
Response:
  { fields: FieldDefinition[] }

// 创建字段
POST /api/fields
Body:
  {
    field_name: string,
    field_type: string,
    options?: string[]
  }
Response:
  { field: FieldDefinition }

// 更新字段
PUT /api/fields/:id
Body:
  {
    field_name?: string,
    options?: string[],
    is_visible?: boolean
  }
Response:
  { field: FieldDefinition }

// 删除字段（软删除）
DELETE /api/fields/:id
Response:
  { success: boolean }
```

### 发布管理 API

```typescript
// 发布到平台
POST /api/works/:id/publish
Body:
  {
    platforms: string[],  // ['douyin', 'xiaohongshu']
    scheduled_at?: string
  }
Response:
  { publish_logs: PublishLog[] }

// 获取发布状态
GET /api/works/:id/publish-status
Response:
  {
    logs: PublishLog[]
  }
```

### 数据追踪 API

```typescript
// 获取作品数据
GET /api/works/:id/analytics
Query:
  - platform?: string
  - days?: number (1 | 3 | 7 | 30)
Response:
  {
    platforms: {
      [platform: string]: {
        day1: Metrics,
        day3: Metrics,
        day7: Metrics,
        day30: Metrics
      }
    },
    trend: TrendData[]
  }
```
