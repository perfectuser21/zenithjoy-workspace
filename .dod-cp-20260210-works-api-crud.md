# Definition of Done: Works Management API (CRUD)

## 📋 验收清单

### 1. API 端点实现

#### Works API (5 个端点)
- [ ] **GET /api/works** - 获取作品列表
  - Test: 支持 type/status/account/limit/offset 参数
  - Test: 返回正确的分页数据和总数
  - Test: 默认按 created_at DESC 排序

- [ ] **GET /api/works/:id** - 获取单个作品
  - Test: 返回完整的作品对象
  - Test: 不存在的 ID 返回 404

- [ ] **POST /api/works** - 创建作品
  - Test: 成功创建返回 201 和作品对象
  - Test: 缺少必填字段返回 400
  - Test: 自动生成 UUID 和时间戳

- [ ] **PUT /api/works/:id** - 更新作品
  - Test: 成功更新返回更新后的对象
  - Test: 不存在的 ID 返回 404
  - Test: updated_at 自动更新

- [ ] **DELETE /api/works/:id** - 删除作品
  - Test: 成功删除返回 200
  - Test: 不存在的 ID 返回 404
  - Test: 级联删除关联的 publish_logs

#### Field Definitions API (4 个端点)
- [ ] **GET /api/fields** - 获取所有字段定义
  - Test: 返回所有可见字段
  - Test: 按 display_order 排序

- [ ] **POST /api/fields** - 创建字段
  - Test: 成功创建返回 201
  - Test: field_name 必须唯一
  - Test: 验证 field_type 有效性

- [ ] **PUT /api/fields/:id** - 更新字段
  - Test: 成功更新返回更新后的对象
  - Test: field_name 唯一性检查

- [ ] **DELETE /api/fields/:id** - 删除字段
  - Test: 成功删除返回 200
  - Test: 不存在的 ID 返回 404

#### Publish Logs API (3 个端点)
- [ ] **GET /api/works/:workId/publish-logs** - 获取发布日志
  - Test: 返回指定作品的所有发布日志
  - Test: work_id 不存在返回空数组

- [ ] **POST /api/works/:workId/publish-logs** - 创建发布日志
  - Test: 成功创建返回 201
  - Test: (work_id, platform) 唯一性约束

- [ ] **PUT /api/publish-logs/:id** - 更新发布状态
  - Test: 成功更新返回更新后的对象
  - Test: 验证 status 有效性

## 🧪 测试方法

### 本地测试

```bash
# 1. 安装依赖
cd apps/api
npm install

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env 填入数据库配置

# 3. 运行测试
npm test

# 4. 启动服务
npm run dev

# 5. 手动测试 API
curl http://localhost:5200/api/works
```

