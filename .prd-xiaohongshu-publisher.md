## 📊 小红书内容类型分析

### 1. 图文笔记（Image Note）

**特点**：
- 最多 9 张图片（1:1 方形或 3:4 竖版）
- 标题 + 正文（支持话题标签 #）
- 支持添加位置信息
- 图片顺序可调整

**必需字段**：
- 图片（至少 1 张，最多 9 张）
- 标题（必需）
- 正文（可选，但推荐填写）

**可选字段**：
- 话题标签
- 位置信息
- 发布时间（定时发布）
- 合集

### 2. 视频笔记（Video Note）

**特点**：
- 单个视频文件
- 自动生成封面（可手动选择）
- 标题 + 正文
- 视频时长：15秒 - 5分钟

**必需字段**：
- 视频文件
- 标题
- 封面（自动或手动）

**可选字段**：
- 正文
- 话题标签
- 位置信息
- BGM（背景音乐）

## 🏗️ 技术实现方案

### 架构

```
NAS 存储
    ↓ 扫描 xhs-post-* 目录
Mac mini 调度器
    ↓ scheduler-xhs.sh
    ↓ 生成 queue.json
    ↓ Base64 编码 + scp 传输
Windows PC
    ↓ publish-xiaohongshu-image.js
    ↓ publish-xiaohongshu-video.js
    ↓ Playwright CDP 连接
小红书创作中心 ✅
```

### 关键技术点

| 技术 | 实现方式 | 参考 |
|------|---------|------|
| **浏览器连接** | CDP 连接 localhost:19222 | 头条/抖音 |
| **文件上传** | `setInputFiles()` | 头条/抖音 |
| **多图上传** | 数组传递给 `setInputFiles()` | 抖音图文 |
| **成功判断** | API 监听 + URL 跳转 | 头条/抖音 |
| **JSON 传输** | Base64 编码 | 头条/抖音 |
| **文件传输** | scp 通过 Mac mini | 头条/抖音 |

### 脚本设计

**文件位置**: `C:\Users\xuxia\playwright-recorder\`

| 脚本 | 功能 | 预计大小 |
|------|------|----------|
| `publish-xiaohongshu-image.js` | 图文笔记发布 | ~4 KB |
| `publish-xiaohongshu-video.js` | 视频笔记发布 | ~4 KB |

**核心流程**（图文笔记）:
```javascript
1. 连接浏览器: chromium.connectOverCDP('http://localhost:19222')
2. 导航到发布页: goto('https://creator.xiaohongshu.com/...')
3. 上传图片: fileInput.setInputFiles(images)
4. 等待上传: waitForTimeout(10s)
5. 填写标题: titleInput.fill(title)
6. 填写正文: contentInput.fill(content)
7. 监听 API: page.on('response', ...)
8. 点击发布: publishBtn.click()
9. 等待响应: waitForResponse(...)
10. 验证成功: note_id 存在
```

## ⚠️ 风险与挑战

### 已知风险

| 风险 | 影响 | 缓解方案 |
|------|------|---------|
| **小红书反爬虫** | 发布可能失败 | 使用已登录浏览器，模拟真实用户操作 |
| **图片格式限制** | 上传失败 | 预处理图片（压缩、转换） |
| **视频编码问题** | 上传失败 | 使用标准 H.264 编码 |
| **发布限流** | 短时间内无法发布 | 调度器控制发布频率 |
| **封面提取失败** | 视频笔记无封面 | 手动提供 cover.jpg |

### 未知挑战

1. **小红书发布页面结构**（待第一步调研）
2. **成功 API 的准确端点**（待监听确认）
3. **图片顺序是否保持**（待测试验证）
4. **视频上传时间**（取决于文件大小）

## 💡 核心原则

基于头条和抖音的成功经验：

1. **不要假设字段可选性** - 即使界面显示可选，也可能导致失败
2. **优先使用精确选择器** - XPath > 文本选择器
3. **充足的等待时间** - 文件上传需要足够时间
4. **URL 跳转是可靠信号** - 比 API 监听更稳定
5. **手动验证先行** - 先手动测试流程，再写脚本
6. **保持架构一致性** - 复用 NAS → Mac mini → Windows 模式
