# Task Card: publisher 支持 --content-id 从 works 表查内容发布

## 需求（What & Why）

**功能描述**：创建 `publish-by-content-id.sh` 编排脚本，接收 `--content-id <id>` 参数，查询 `zenithjoy.works` 表获取 NAS 路径，从 NAS 读取内容文件，调用平台发布脚本，最后写入 `zenithjoy.publish_logs`。

**背景**：
- `zenithjoy.works` 表已有 `content_id`、`nas_path`、`status` 字段（PR #64）
- 现有 publisher 脚本只支持本地 JSON 文件路径，无法通过 content-id 查 DB
- longform-creator Step 5.5 已将内容注册到 works 表（content-id = NAS 目录名）
- 需要一个"中间层"脚本打通：works DB → NAS 读取 → publisher

**不做什么**：不修改现有 publisher CJS 脚本内部逻辑（向后兼容）。

## 成功标准

1. [ARTIFACT] `scripts/publish-by-content-id.sh` 存在
2. [ARTIFACT] 脚本支持 `--content-id` 和 `--platform` 参数
3. [ARTIFACT] 脚本查询 `zenithjoy.works` 表（使用 psql）
4. [ARTIFACT] 发布完成后写入 `zenithjoy.publish_logs`（INSERT SQL）
5. [BEHAVIOR] 脚本在 works 表无该 content-id 时正常报错退出

## 实现方案

**文件**：`scripts/publish-by-content-id.sh`（zenithjoy repo 根目录下 scripts/）

**流程**：
```
--content-id <id> --platform <kuaishou|weibo|wechat>
    ↓
1. psql: SELECT id, title, nas_path, status FROM zenithjoy.works WHERE content_id = $1
    ↓
2. 验证 status = 'ready'，否则报错
    ↓
3. scp 从 NAS 下载图片到 /tmp/content-{id}/images/
4. cat NAS exports/title.txt → TITLE
5. 生成临时 publish.json {content: TITLE, images: [...]}
    ↓
6. 调用对应 publisher 脚本
    ↓
7. psql: INSERT INTO zenithjoy.publish_logs (work_id, platform, status, published_at)
8. psql: UPDATE zenithjoy.works SET status='published' WHERE content_id = $1
```
