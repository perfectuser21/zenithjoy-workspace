## PRD - 修改 CI 安全审计级别

**需求来源**: CI/CD 专家任务 - 加固 CI/CD 安全配置

**功能描述**:
将 GitHub Actions workflow 中的 npm audit 级别从 `high` 改为 `moderate`，以便更早发现安全漏洞。

当前发现 2 个 moderate 级别漏洞：
1. esbuild <=0.24.2 - 允许任何网站向开发服务器发送请求并读取响应
2. vite 0.11.0 - 6.1.6 - 依赖有漏洞的 esbuild

**涉及文件**:
- `.github/workflows/ci.yml` - 修改第 68 行的 audit-level 参数

**成功标准**:
1. `.github/workflows/ci.yml` 中的 `--audit-level=high` 改为 `--audit-level=moderate`
2. CI 能正常运行并检测到 moderate 级别漏洞
3. 记录当前的 moderate 级别漏洞清单

**非目标**:
- 不修复漏洞本身（需要单独的 PR）
- 不修改其他 CI 配置
