# 今日头条自动发布测试总结

## 测试范围
Test-01 至 Test-06，共6轮测试

## 成功的部分 ✅

1. **连接 Chrome CDP** - 可以通过 WebSocket 连接到远程 Chrome（100.97.242.124:19225）
2. **填写标题** - 使用 React 方法成功填写：
   ```javascript
   const setter = Object.getOwnPropertyDescriptor(window.HTMLTextAreaElement.prototype, 'value').set;
   setter.call(input, '标题内容');
   input.dispatchEvent(new Event('input', {bubbles: true}));
   ```
3. **填写内容** - 直接设置 `.ProseMirror` 的 innerHTML
4. **点击"预览并发布"按钮** - 可以找到并点击红色 primary 按钮
5. **进入封面选择页面** - 点击后确实会跳转到封面配置页面

## 失败的部分 ❌

1. **字数不够**
   - 标题要求: 15字，实测只写了 6-12字
   - 内容要求: 30字，实测只写了几个字
   
2. **封面图片上传失败**
   - 尝试用 `DOM.setFileInputFiles` 但未成功
   - 文件输入框找到了但设置文件后没有触发上传
   
3. **页面状态不稳定**
   - 点击"预览并发布"后，有时进入封面选择页面
   - 有时页面刷新回空白编辑状态
   - 草稿会保存但页面会重置

4. **截图经常超时**
   - `Page.captureScreenshot` 频繁超时
   - 可能是页面加载慢或 WebSocket 连接问题

## 关键发现 🔍

### 发布流程

```
编辑页面
  ↓ 填写标题+内容
  ↓ 点击"预览并发布"
封面选择页面
  ↓ 选择封面（展示封面/单图/三图/无封面）
  ↓ 配置其他设置（合集、位置、技改广告等）
  ↓ 点击最终发布按钮（未找到）
发布成功？
```

### Network 监控发现

- 点击"预览并发布"会调用 `/article/publish` API
- 响应: `{"message": "保存成功"}` 不是 "发布成功"
- 说明这个按钮只是**保存草稿**，不是真正发布

### 按钮位置

- "预览并发布"按钮在页面底部
- 位置: top=548px（窗口高度600px）
- className 包含 'primary'（红色按钮）

## 技术难点 ⚠️

1. **CDP 文件上传**
   - `DOM.setFileInputFiles` 设置文件后没有触发实际上传
   - 可能需要额外触发 change 事件或点击上传按钮

2. **页面加载时机**
   - Page.reload 后需要等待足够长时间（5秒+）
   - 否则后续操作会超时

3. **WebSocket 连接稳定性**
   - 长时间运行会出现超时
   - 可能需要增加 maxListeners 或重新连接

## 下一步建议 📋

### 必须修复

1. **增加字数**
   - 标题: 至少 15 个汉字
   - 内容: 至少 30 个汉字
   - 使用真实内容，不要带 "test" 等测试词

2. **解决图片上传**
   方案A: 找到页面上的图片文件，复制一个到 /tmp/
   方案B: 用 Python PIL 生成简单图片（已做：/tmp/cover-image.jpg）
   方案C: 选择"无封面"选项跳过上传

3. **找到最终发布按钮**
   - 在封面选择页面滚动到最底部
   - 查找包含"发布"或"确认"的 primary 按钮
   - 点击并监控 Network 响应

### 优化建议

1. 减少 Page.reload，直接操作当前页面
2. 增加 WebSocket 超时时间到 10秒
3. 每个操作后等待 2-3 秒确保生效
4. 用 try-catch 包裹截图操作，失败不中断流程
