# 视频号图文 vs 视频字段分析 PRD

## 背景

已完成抖音、快手、小红书的图文 vs 视频字段分析，发现三种不同的架构模式：
- **抖音**：分开 API，字段完全不同
- **快手**：统一 API，字段完全相同
- **小红书**：统一 API，字段部分重叠

现在需要分析**视频号**（微信视频号）平台的图文和视频内容类型差异。

## 目标

分析视频号平台的图文和视频作品，提取完整的字段清单，明确两者的区别。

## 产出物

1. **字段对比表**：
   - 视频完整字段列表（含字段名、示例值、说明）
   - 图文完整字段列表（含字段名、示例值、说明）
   - 标注专属字段和共同字段

2. **区分规则**：
   - 如何在页面/API中区分视频和图文
   - 关键标识字段（如抖音的"X张"、快手的时长）

3. **数据来源**：
   - 管理页面URL
   - 统计/分析页面URL
   - API端点（如果有）

4. **采集策略**：
   - 推荐的采集方式（API/DOM/其他）
   - 是否需要分别处理视频和图文
   - 与已有采集器 v3 的对比和升级建议

## 技术细节

### 已知信息

**视频号创作者平台**：
- URL: `https://channels.weixin.qq.com/`
- 浏览器端口: `100.97.242.124:19228`
- 当前采集器: `/home/xx/scraper-channels-v3.js`

**已知特征**：
- 视频号是微信生态内的内容平台
- 支持视频和图文（图片）内容
- 有创作者后台数据分析功能

### 分析步骤

1. **页面导航**：
   - 访问视频号创作者后台
   - 找到作品管理/数据分析页面
   - 确认页面正确加载

2. **内容类型识别**：
   - 查找视频作品示例
   - 查找图文作品示例
   - 识别区分标识（图标、标签、字段）

3. **API 拦截**：
   - 监听网络请求
   - 识别数据 API 端点
   - 捕获完整的 API 响应
   - 分析 JSON 结构

4. **字段提取**：
   - 截图作品列表
   - 提取 API 响应中的所有字段
   - 对比视频和图文的字段差异

5. **完播率数据**：
   - 确认视频是否有完播率
   - 确认图文是否有完播率（或类似指标）
   - 找到数据来源（DOM/API）

6. **对比分析**：
   - 制作字段对比表
   - 标注专属字段
   - 归类到三种架构模式（分开API/统一相同/统一重叠）
